## 涉及页面

### 1. pages/index/index.vue (改造已有)
**位置**: 在现有入口卡片下方追加一个"服务"区块
**新增 2 个入口**:
- 「赛事方入驻」icon=🏛️ → 跳 `pages-sub/public/partner-apply`
- 「联系客服」icon=💬 → 跳 `pages-sub/public/help`
**视觉**: 与已有入口卡片样式一致,2 列布局

### 2. pages-sub/public/partner-apply.vue (新建) ⭐
**对应原型**: 自由设计(原型未提供完整页,可参考 teacher-identity.html / event-apply.html 的表单样式)
**结构**:
- 顶部 banner: "竞赛通赛事方入驻"
- 表单字段(对照后端 PartnerApplyReqDTO):
  - 机构名称 (必填,@NotBlank, max 128)
  - 联系人姓名 (必填)
  - 联系手机 (必填,11 位,正则校验)
  - 营业执照号 (可选)
  - 资质材料图片上传 (可选,最多 5 张)
  - 简单结算/发票备注信息(可选,文本框)
- 同意协议 checkbox
- 提交按钮 → 调 POST /jst/public/organizer/partner/apply
- 提交成功 → 显示申请单号 + 跳"查询状态"页

### 3. pages-sub/public/apply-status.vue (新建)
**用途**: 输入申请单号查询审核进度
**结构**: 单 input 输入 applyNo + 查询按钮 → 调 GET status 接口 → 展示
**可重入**: 用户可多次查不同单号

### 4. pages-sub/public/help.vue (新建)
**对应原型**: `小程序原型图/help.html`
**结构** (静态页,无 API):
- 常见问题 FAQ (折叠面板,5-10 条预设)
- 联系方式 (客服电话/微信号/邮箱)
- 用户协议 / 隐私政策入口 (跳本页内 anchor 或新页)
**数据**: 全部硬编码 in template (本任务唯一允许的硬编码,因为静态 FAQ 不是业务数据)

> 注: 这里的"硬编码"指 FAQ 文本,不违反 25 §3.8 (该规约禁止的是**业务数据** mock,如订单/赛事列表)。FAQ 是 UI 文案,合法。

## 测试验证 (本地浏览器)

### P5-1 首页入口
1. 启动 jst-uniapp,登录后切到首页
2. 应能看到"赛事方入驻"和"联系客服"两个新入口

1. 点"赛事方入驻"

### P5-4 帮助页
1. 点"联系客服"
2. 应看到 FAQ 折叠 + 联系方式
