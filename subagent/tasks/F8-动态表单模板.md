## 业务背景
赛事报名表单字段是**动态可配**的。每个赛事可关联一个表单模板,模板由 schema_json 描述字段定义。
- 表单模板有版本号,**修改即递增**
- 报名时**锁定**当前版本到 enroll_record.form_snapshot_json (F9 任务做)
- 模板审核后才可被赛事使用

**为什么独立做 F8**:F7 赛事和 F9 报名都依赖它,作为中间件优先实现

## 涉及状态机
**SM-25** `jst_enroll_form_template.audit_status`:
```
draft → pending → (approved | rejected)
```
版本变更不影响历史 enroll_record.form_snapshot_json (F9 时强约束)

## 涉及锁
- `lock:form:template:save:{templateId}` (wait 3s, lease 5s) - 保存(自增版本)

## 涉及权限
- 赛事方编辑: `jst:event:formTemplate:edit`
- 平台审核: `jst:event:formTemplate:audit`
- 小程序读取: `@Anonymous` (报名前拉取已通过的模板)

## 接口契约

### 1. POST /jst/event/form/template/save
**用途**: 创建/更新表单模板
**入参**: FormTemplateSaveReqDTO
```java
{
  Long templateId;          // 编辑时填,新建为 null
  @NotBlank String templateName;
  @NotBlank String ownerType;  // platform/partner
  @NotNull Object schemaJson;  // 字段定义(数组), backend 直接 JSON 序列化存储
}
```
**业务**:
1. lock
2. 校验 schemaJson 结构合法 (顶层必须是 {fields:[...]} 或直接是 [...]; 每个 field 必须有 key/type/label/required 字段)
3. 字段类型白名单: text/textarea/radio/checkbox/select/date/age/number/image/audio/video/file/group/conditional
4. 新建: INSERT, version=1, audit_status=draft
5. 编辑: 必须在 draft/rejected 状态; UPDATE schemaJson + version+1, audit_status=draft (改了内容自动转回 draft 等再审核)

### 2. POST /jst/event/form/template/{id}/submit
**业务**: 状态机 draft→pending

### 3. POST /jst/event/form/template/{id}/audit/approve
**入参**: AuditReqDTO {String auditRemark}
**业务**: pending→approved, effective_time=now()

### 4. POST /jst/event/form/template/{id}/audit/reject
**入参**: AuditReqDTO {@NotBlank String auditRemark}
**业务**: pending→rejected

### 5. GET /jst/event/form/template/list
**用途**: admin 列表(含 PartnerScope 过滤 owner_id)
**入参**: FormTemplateQueryReqDTO

### 6. GET /jst/event/form/template/{id}
**用途**: admin 详情(含 schemaJson 完整内容)

### 7. GET /jst/wx/enroll/template ⭐
**用途**: 小程序根据 contestId 取报名表单
**入参**: `@RequestParam Long contestId`
**出参**:
```java
{
  Long templateId;
  Integer templateVersion;
  Object schemaJson;
}
```
**业务**:
1. 查 jst_contest by contestId, 校验 audit_status=online
2. 取 contest.form_template_id
3. 查 template, 校验 audit_status=approved AND status=1
4. 返回完整 schema (前端 jst-form-render 组件用)
**权限**: `@Anonymous`
