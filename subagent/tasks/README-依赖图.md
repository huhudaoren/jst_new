完成后,小程序应能跑通:
- 登录 → 我的 → 编辑资料 → 临时档案

- 公告 tab 浏览 + 详情

**总工时**: ~50-60 小时 (3 个 Backend Agent 并行 + 1 个 Frontend Agent 串行约 1-2 周)

## 共享文件冲突处理 (主 Agent 合并)

3 个 Backend Agent 并行时这些文件可能冲突:
- `BizErrorCode.java` - 各自追加错误码 (不同段位,理论不冲突,但可能格式不一致)
- `99-test-fixtures.sql` - 各自追加 fixture 数据
- `api-tests.http` - 各自追加 ### 测试块

**子 Agent 完成后,把所有报告一起发给我,我做合并**。
不要让某个子 Agent 同时改另一个 Agent 改过的文件。

---

## Frontend 与 Backend 协作约定

P1 / P2 / P3 / P4 都是 Frontend Agent 任务,**严格禁止改后端代码**。
F* 都是 Backend Agent 任务,**严格禁止改前端代码** (jst-uniapp 目录)。

跨工程的协作通过**接口契约** (27 文档) 完成,前后端只通过 .http 测试 + Postman 联调,**不直接依赖对方源码**。

如发现接口契约不清,**前端 Agent 应停止并报告**给主 Agent,由主 Agent 决定是更新文档还是让 Backend Agent 修接口。

---

## 重要提醒

如果 Backend Agent 在做 F-NOTICE 时发现需要新建表 / 改 ddl → **必须停止**,把需求告诉主 Agent,由主 Agent 写 migration sql 后再继续。
