## 业务背景
学生在小程序选定参赛人 + 填动态表单 → 提交报名 → 写入 jst_enroll_record。
**本任务不创建订单**(订单在 C2 单独 feature 做)。

## 涉及状态机
**SM-6** `jst_enroll_record.audit_status`:
```
pending → (approved | rejected | supplement)
supplement → pending
```

并存的 **material_status**:
```
draft → submitted → supplement
```
(用户视角的资料状态,与 audit_status 解耦)

## 涉及锁
- `lock:enroll:submit:{userId}:{contestId}` (wait 3s, lease 5s) - 防同一用户同一赛事并发重复提交
  > 必须先在 `15-Redis-Key与锁规约.md` 追加登记(由主 Agent 后续合并)

## 涉及权限
- 学生端: `hasRole('jst_student')` + 归属校验 (user_id = current)
- 赛事方端: `jst:event:enrollRecord:list/query/audit` + `@PartnerScope` (只能看自己赛事的报名)

## 防雷点 (主 Agent 强约束)

1. **form_snapshot 一旦写入 immutable**:
   - 提交时复制当前 template 的 schema_json + version 到 enroll_record.form_snapshot_json
   - 后续 template 修改不影响历史 enroll_record (form_template 服务保证)
   - 本任务的 update enroll_record 不允许覆盖 form_snapshot_json 字段

2. **同一用户同一赛事不允许重复 pending**:
   - 提交前查 `WHERE user_id=? AND contest_id=? AND audit_status IN (pending, supplement)`
   - 已有则抛 DUPLICATE 错
   - 已 approved 的允许补报 (可能 PRD 后期改,先按允许处理)

3. **参赛人归属校验**:
   - 入参 participant_id 必须满足: 该 participant 已被当前 user 认领 (jst_participant_user_map.user_id=current 且 status=active)
   - 防越权: A 用 B 的子女档案报名

4. **赛事必须在报名期 + online**:
   - 校验 contest.audit_status='online' AND contest.status='enrolling' AND now() BETWEEN enroll_start_time AND enroll_end_time
   - 否则抛 NOT_ENROLLING

5. **本任务不创建订单**:
   - enroll_record.order_id = NULL
   - C2 任务里订单创建时反向回填 order_id 到 enroll_record

6. **草稿 (draft) 与正式提交分离**:
   - draft 接口不做严格校验 (允许部分字段为空)
   - submit 接口才做严格校验 + 锁

## 接口契约 (4 wx + 3 admin)

### W1. POST /jst/wx/enroll/draft
**用途**: 保存草稿
**入参**:
```java
class EnrollDraftDTO {
    @NotNull Long contestId;
    @NotNull Long participantId;
    Map<String,Object> formData;  // 表单字段值,允许部分为空
    List<String> attachments;     // 已上传文件 URL 列表 (来自 OssService)
    Long enrollId;                // 编辑已有草稿时填,新建为 null
}
```
**出参**: `AjaxResult<{enrollId, enrollNo}>`
**权限**: `hasRole('jst_student')`
**业务**:
1. 校验 contest 在报名期(可宽松,允许提前存草稿)
2. 校验 participant 归属当前 user
3. 拉取 contest.form_template_id 对应模板的当前 version
4. UPSERT enroll_record:
   - new: INSERT, material_status=draft, audit_status=pending (草稿也算 pending)
   - edit: UPDATE 字段,version 重新拉(允许草稿期间模板版本变化)
5. form_snapshot_json = `{template_id, template_version, formData, attachments}`

### W2. POST /jst/wx/enroll/submit ⭐ 核心
**用途**: 正式提交报名
**入参**:
```java
class EnrollSubmitDTO {
    @NotNull Long contestId;
    @NotNull Long participantId;
    @NotNull Map<String,Object> formData;
    List<String> attachments;
    Long enrollId;                // 从 draft 转 submit 时填
}
```
**出参**: `AjaxResult<{enrollId, enrollNo}>`
**权限**: `hasRole('jst_student')`
**业务流程** (lock + tx):
1. lock `lock:enroll:submit:{userId}:{contestId}`
2. 校验 contest: audit_status=online + status=enrolling + 当前时间在报名期
3. 校验 participant 归属当前 user (jst_participant_user_map active)
4. 校验 schema 字段 (后端按模板的 required 字段做最小校验,具体校验由前端 jst-form-render 已做)
5. 查重: WHERE user_id=current AND contest_id=current AND audit_status IN (pending, supplement)
   - 已有 enroll_id != 入参 enroll_id → 抛 DUPLICATE
   - enroll_id == 入参 → 是 draft 转 submit,UPDATE
6. UPSERT: material_status=submitted, audit_status=pending, submit_time=now()
7. form_snapshot_json **锁定** (从此不可改)
8. 写 audit_log
9. 返回 enrollId

### W3. GET /jst/wx/enroll/{id}
**用途**: 查单条报名详情
**出参**: `EnrollDetailVO` (含 contest 名称冗余 + 当前 audit_status + form_snapshot_json)
**权限**: `hasRole('jst_student')` + Service 层 SecurityCheck.assertSameUser(enroll.user_id)

### W4. POST /jst/wx/enroll/{id}/supplement
**用途**: 补充材料(状态机 supplement → pending)
**入参**:
```java
class EnrollSupplementDTO {
    @NotNull Map<String,Object> formData;
    List<String> attachments;
    String supplementRemark;
}
```
**业务**:
1. 校验 enroll 归属当前 user
2. 校验 audit_status='supplement' (其他状态不可补充)
3. SM-6 跃迁 supplement → pending
4. UPDATE form_snapshot_json (注:补充材料阶段允许覆盖,因为还未通过)
5. UPDATE submit_time, audit_remark 清空

### A1. GET /jst/event/enroll/list (admin)
**入参**: `EnrollQueryReqDTO {Long contestId, String auditStatus, String materialStatus, ...}`
**出参**: `TableDataInfo<EnrollListVO>`
**权限**: `jst:event:enrollRecord:list` + `@PartnerScope` (赛事方只能看自己赛事的报名)

### A2. GET /jst/event/enroll/{id} (admin)
**用途**: 详情(含完整 form_snapshot_json + 参赛人脱敏信息)
**权限**: `jst:event:enrollRecord:query` + Service 校验 contest.partner_id 归属

### A3. POST /jst/event/enroll/{id}/audit (admin) ⭐
**用途**: 赛事方审核
**入参**:
```java
class EnrollAuditReqDTO {
    @NotBlank @Pattern(regexp="approved|rejected|supplement") String result;
    String auditRemark;
}
```
**业务流程** (lock + tx):
1. lock `lock:enroll:audit:{enrollId}`
2. 校验 enroll 归属当前 partner (contest.partner_id=current)
3. SM-6 跃迁 pending → result
4. UPDATE audit_status, audit_remark
5. result=supplement 时不做额外动作(等用户补充)
6. result=approved 时**不**自动创建订单(订单由 C2 通过其他流程创建)
7. 写 audit_log + 触发 mock 消息通知
