# 任务卡 C4 - 退款全流程 (CCB ⭐⭐ 高风险)

## 🔴 这是阶段 C 第二高风险 feature (仅次于 C2)

涉及**资金逆向回退** + **多表状态联动** + **返点台账联动**。
任何 bug 都可能导致用户钱拿不回来,或重复退款,或返点台账错乱。

## CCB 决策点 (派发前 user 过目)

| # | 决策 | 选择 |
|---|---|---|
| 1 | **谁可发起退款** | 学生用户 (售后期内) + 平台管理员 (特批,过期也可) |
| 2 | **是否需要赛事方审核** | **是**,默认所有退款先经赛事方,赛事方驳回再走平台特批 |
| 3 | **全额 vs 部分退款** | 本期**仅支持全额退款**,部分退款留给后期 (订单 quantity=1 不需要部分退) |
| 4 | **资金回退顺序** (SM-2 强制) | 现金 → 积分 → 优惠券 |
| 5 | **优惠券回退条件** | **仅**整单全退 + 原券**有效期内** + 原券**未被消耗后又用过** |
| 6 | **报名联动** | 退款 completed 后 enroll_record.audit_status='cancelled' (新增枚举) + order_id 不清(留作记录) |
| 7 | **订单联动** | 退款 completed → order.refund_status='full' + order_status='aftersale'→'cancelled' |
| 8 | **返点联动** | 未计提 → ledger 直接 rolled_back;已计提未打款 → rolled_back;已打款 → 新增 negative 一行 |
| 9 | **微信支付退款** | Mock 模式直接 success;真实模式留 RealWxPayServiceImpl.refund 占位 |
| 10 | **退款幂等** | 同一 refund_no 重复 execute 仅生效一次,wxpay refund 回调防重 |
| 11 | **零元订单退款** | 直接 reject (没钱可退,不允许发起) |

## 涉及表

**写表**:
- jst_refund_record (主)
- jst_order_main (refund_status / order_status 联动)
- jst_order_item (refund_amount / refund_points 累加)
- jst_rebate_ledger (写 rolled_back / 新增 negative)

**联动写**:
- jst_user / jst_points_account (积分回退到 available)
- jst_points_ledger (写 rollback 流水)
- jst_user_coupon (used → unused 或保持 used,看条件)
- jst_enroll_record (audit_status → cancelled, 通过跨域 SQL)

**只读校验**:
- jst_payment_record (取支付方式判断 wxpay/bank/points)

## 涉及状态机

**SM-2 jst_refund_record.status**:
```
pending → approved → refunding → completed
   ↓           ↓
rejected    rejected
   ↓           ↓
 closed     closed
```

**联动 SM-1 (退款 completed 后)**:
```
in_service → aftersale → cancelled
```

**联动 SM-8 rebate_ledger.status**:
```
pending_accrual → rolled_back   (未计提的直接置回)
withdrawable    → rolled_back   (已计提未提现)
paid            → 不动 + 新增一条 negative ledger
```

## 涉及锁

| 锁名 | 用途 | wait/lease |
|---|---|---|
| `lock:refund:apply:{orderId}` | 防同一订单并发申请退款 | 3s / 5s |
| `lock:refund:approve:{refundId}` | 审核操作并发保护 | 3s / 5s |
| `lock:refund:execute:{refundId}` | 执行退款资金回退 | 5s / 30s ⭐ 长事务 |
| `lock:wxpay:refund:notify:{refundNo}` | 微信退款回调防重 | 5s / 10s |

## 防雷点 (12 条强制清单)

### 1. 资金回退顺序严格
执行退款时必须按照: 现金 → 积分 → 优惠券。**禁止**调换或并行。

### 2. 一个大事务覆盖资金回退
`RefundService.execute()` 必须 `@Transactional(rollbackFor = Exception.class)`,内部:
1. 状态机校验 SM-2 approved → refunding
2. 调微信支付退款接口 (mock 直接 success)
3. UPDATE order_main: refund_status, order_status
4. UPDATE order_item: refund_amount, refund_points 累加
5. INSERT points_ledger 回退流水 + UPDATE points_account: available += pointsRefund
6. UPDATE user_coupon (满足回退条件时): status='unused'
7. UPDATE rebate_ledger 状态推进 (按 9 不同情况)
8. 跨域 UPDATE enroll_record.audit_status='cancelled'
9. 状态机 SM-2 refunding → completed
10. 写 audit_log

任何一步失败 → 整事务回滚 → refund_record.status 不前进 → 用户可重试

### 3. 状态机校验全部用 assertCanTransitTo
**禁止** if-else 散落,必须用枚举状态机。

### 4. 乐观锁更新主状态
```sql
UPDATE jst_refund_record SET status='completed', complete_time=NOW()
WHERE refund_id = ? AND status = 'refunding'
-- updated == 0 抛 OPTIMISTIC_LOCK
```

### 5. 优惠券回退条件严格
仅当**所有**条件满足才回退 status='unused':
- 整单全退 (refund_type='refund_only' AND apply_cash 等于 order.net_pay)
- 原券 valid_end > now()
- order_main.coupon_id 不为 null
否则**保持 used**,coupon_returned=0

### 6. 已打款的返点必须新增 negative ledger
**不要** UPDATE 原 paid 记录(它是真实的发生过的提现)。
INSERT 一行 direction='negative' rebate_amount=负数,关联同 channel_id。
后续 J05 计提 Job + 渠道下次提现时由 SM-9 approve 阶段做负向抵扣。

### 7. 报名状态联动 + 跨域 SQL
本任务**不能 import** jst-event 的 Entity (违反模块边界)。
新增 `EnrollLookupMapper` 在 jst-order 内 (类似已有 lookup mapper),仅 UPDATE jst_enroll_record SET audit_status='cancelled' WHERE enroll_id IN (订单的 enroll_id 列表)。

需要在 EnrollAuditStatus 枚举追加 CANCELLED 值 (允许 approved → cancelled 跃迁)。
**或**: 不改 enroll 状态机,只在 enroll_record 上加冗余字段 cancelled_by_refund? 决策:**改状态机加 CANCELLED**,因为业务清晰。

### 8. 零元订单退款拒绝
service 入口校验 order.list_amount=0 → 抛 NO_REFUND_FOR_ZERO_PRICE

### 9. 退款金额校验
申请阶段: apply_cash + apply_points*pointsCashRate <= order.net_pay (本期=,因仅全退)
执行阶段: actual_cash/actual_points 由后端重算,不信前端

### 10. 微信支付退款幂等
RealWxPayServiceImpl.refund 留空 throw NotImplemented (本期 mock)
MockWxPayServiceImpl.refund 直接返回 success + 记录 mock log
真实回调 /jst/wx/pay/refund-notify 必须 lock + status check 幂等

### 11. 退款时机限制
- 学生发起: order.aftersale_deadline > now() (售后期内)
- admin 特批: 任意时间允许 (跳过 deadline 校验)
- 所有发起方: order.allow_self_refund=1 才允许学生自助退,否则只能 admin 发起

### 12. 重复退款防护
- 同一 order_id 不允许有多条 status in (pending, approved, refunding) 的 refund_record
- 已 completed 的 refund_record 也算占位 (因本期仅全退,退完就不能再退)

## 接口契约 (4 wx + 4 admin)

### W1. POST /jst/wx/order/{orderId}/refund (学生申请退款)
**入参**:
```java
class RefundApplyDTO {
    @NotBlank @Pattern(regexp="refund_only|return_refund") String refundType;
    @NotBlank @Size(max=255) String reason;
}
```
**出参**: `AjaxResult<{refundId, refundNo}>`
**权限**: `hasRole('jst_student')` + 订单归属校验
**业务**:
1. lock `lock:refund:apply:{orderId}`
2. 校验 order 归属 + 状态 in (paid, in_service, aftersale)
3. 校验 order.allow_self_refund=1
4. 校验 order.aftersale_deadline > now()
5. 校验 list_amount > 0
6. 校验无重复 refund 占位
7. INSERT refund_record: status=pending, apply_cash=order.net_pay, apply_points=order.points_used, refund_type=user 选
8. UPDATE order_main: order_status → aftersale (状态机)
9. 返回 refundId

### W2. GET /jst/wx/refund/{refundId} (学生查退款详情)
**权限**: 归属校验

### W3. GET /jst/wx/refund/my (我的退款列表)
**权限**: hasRole('jst_student'),自动过滤 user_id

### W4. POST /jst/wx/refund/{refundId}/cancel (学生撤销退款申请)
**业务**: 仅 pending 状态可撤销,UPDATE status='closed' + UPDATE order_main 回退到原状态
**权限**: 归属校验 + 仅 pending

### A1. GET /jst/order/refund/list (admin 查退款列表)
**权限**: jst:order:refund:list + 数据权限

### A2. POST /jst/order/refund/{refundId}/approve (赛事方/平台审核通过)
**入参**: `{auditRemark}`
**业务**: lock + SM-2 pending → approved + UPDATE remark
**权限**: jst:order:refund:approve

### A3. POST /jst/order/refund/{refundId}/reject (审核驳回)
**入参**: `{@NotBlank auditRemark}`
**业务**: SM-2 pending → rejected + UPDATE 联动 order 回到原状态

### A4. POST /jst/order/refund/{refundId}/execute (执行退款资金回退) ⭐⭐ 核心
**入参**: 无 (后端基于 refund_record 重算)
**业务**: 大事务 (见防雷点 #2)
**权限**: jst:order:refund:execute (admin 才有,赛事方无此权限)

### A5. POST /jst/wx/pay/refund-notify/wechat (微信退款回调,公开)
**权限**: @Anonymous + 验签 (mock 跳过)
**幂等**: 必须

### A6. POST /jst/admin/order/{orderId}/special-refund (admin 特批退款)
**用途**: 跳过售后期 / 跳过赛事方审核
**权限**: jst:order:refund:special

## 退款执行算法 (强制按此实现)

```java
@Transactional(rollbackFor = Exception.class)
public void execute(Long refundId) {
    RefundRecord refund = lock("lock:refund:execute:" + refundId, () -> {
        // 1. 校验 + 状态机
        RefundRecord r = mapper.selectById(refundId);
        RefundStatus.fromDb(r.getStatus()).assertCanTransitTo(RefundStatus.REFUNDING);
        OrderMain order = mapper.selectOrderById(r.getOrderId());
        
        // 2. 调微信退款 (mock 模式直接 success)
        WxRefundResult wxRes = wxPayService.refund(buildRefundReq(r, order));
        
        // 3. UPDATE refund_record refunding
        updateStatus(refundId, "approved", "refunding");
        
        // 4. 现金回退 (微信侧已退,后端只记账)
        BigDecimal cashRefunded = order.getNetPayAmount();
        
        // 5. 积分回退
        if (order.getPointsUsed() > 0) {
            pointsLookupMapper.refundPoints(order.getUserId(), order.getPointsUsed());
            pointsLookupMapper.insertLedger(...);
        }
        
        // 6. 优惠券回退 (条件严格)
        if (canReturnCoupon(order, r)) {
            couponLookupMapper.unlockCoupon(order.getCouponId());
        }
        
        // 7. 返点台账回退
        rebateLedgerLookupMapper.rollbackByOrderId(order.getOrderId());
        // 内部根据 status 走 rolled_back 或 negative
        
        // 8. order 状态机 in_service → aftersale → cancelled
        orderMainMapper.updateStatusToCancelled(order.getOrderId());
        
        // 9. order_item 累加 refund_amount/points
        orderItemMapper.accumulateRefund(order.getOrderId(), cashRefunded, order.getPointsUsed());
        
        // 10. 跨域 UPDATE enroll_record.audit_status='cancelled'
        enrollLookupMapper.cancelEnrollByOrderId(order.getOrderId());
        
        // 11. UPDATE refund_record completed
        updateStatus(refundId, "refunding", "completed");
        
        // 12. 写 audit_log
        return r;
    });
}
```

## 测试场景 (12+ 用例)

### 正向流程
1. **C4-1** 用户申请退款 (售后期内)
2. **C4-2** 赛事方审核通过 → admin 执行 → 资金回退
3. **C4-3** 资金账目对账: cash + points + coupon + enroll + rebate 全部正确
4. **C4-4** 学生撤销 pending 退款
5. **C4-5** 赛事方驳回 → 平台特批 → 资金回退

### 异常 / 边界
6. **C4-6** 售后期已过 学生申请退款应拒绝
7. **C4-7** 零元订单不允许退款
8. **C4-8** 重复申请退款应拒绝
9. **C4-9** 已 completed 的退款再次 execute 应幂等
10. **C4-10** 已打款的返点退款应新增 negative 行
11. **C4-11** 优惠券过期后退款不回退券
12. **C4-12** 跨域 enroll.audit_status 应正确变 'cancelled'

### 越权
13. **C4-13** A 申请 B 订单的退款应失败
14. **C4-14** 赛事方调 execute 应 403 (仅 admin)
