# 任务卡 F6 - 渠道方认证申请 + 审核

## 阶段 / 模块
阶段 B / jst-organizer

## 业务背景
普通学生用户可在小程序内申请成为「渠道方」(老师/机构),提交身份认证材料。
平台运营审核通过后:
1. 创建 jst_channel 主档,关联到当前 user_id
2. 给该 user 加一个新角色 jst_channel (保留原 jst_student 角色,**双角色**)
3. 通知用户

与 F5 的区别:
- F5 是**新机构入驻**(从无到有,创建 sys_user)
- F6 是**已有学生升级为渠道方**(复用 sys_user,只加角色)

## 涉及表
- jst_channel_auth_apply (主)
- jst_channel (审核通过后创建)
- sys_user_role (审核通过后追加 jst_channel 角色)

## 涉及状态机
**SM-3** `jst_channel.auth_status` + `jst_channel_auth_apply.apply_status`:
```
pending → (approved | rejected)
approved → suspended  (启停)
```

## 涉及锁
- `lock:org:channelApply:{applyId}` (wait 3s, lease 10s)

## 涉及权限标识
- 用户提交申请: `hasRole('jst_student')`
- admin 审核: `jst:organizer:channelApply:list`, `:approve`, `:reject`

## 接口契约

### 1. POST /jst/wx/channel/auth/apply
**用途**: 当前用户提交认证申请
**入参**:
```java
class ChannelAuthApplyReqDTO {
    @NotBlank @Pattern(regexp="teacher|organization|individual") String channelType;
    @NotBlank @Size(max=128) String applyName;
    @NotBlank String materialsJson;  // 认证材料JSON (身份证/教师资格证等图片URL)
}
```
**出参**: `AjaxResult<{applyId, applyNo}>`
**权限**: `hasRole('jst_student')` (登录即可)
**业务**:
1. 校验当前用户是否已有 status in (pending, approved) 的申请 → 拒重复
2. 校验当前用户是否已经是 channel (jst_channel.user_id = currentUser AND status=1 AND auth_status=approved) → 拒"已是渠道方"
3. INSERT 状态 = pending,生成 apply_no = "CA" + 雪花ID
4. user_id 从 token 取(禁止入参)

### 2. GET /jst/wx/channel/auth/apply/my
**用途**: 当前用户查自己的申请历史
**出参**: `AjaxResult<List<ChannelAuthApplyVO>>`
**权限**: `hasRole('jst_student')`

### 3. (admin) GET /jst/organizer/channel/apply/list
**用途**: 审核员查询
**入参**: `ChannelAuthApplyQueryDTO {String applyStatus, String channelType, ...}`
**权限**: `:list`

### 4. (admin) GET /jst/organizer/channel/apply/{applyId}
**用途**: 详情
**权限**: `:detail`

### 5. (admin) POST /jst/organizer/channel/apply/{applyId}/approve  ⭐ 核心
**用途**: 通过审核
**入参**: `ApproveChannelReqDTO {String auditRemark, String tags}`
**出参**: `AjaxResult<{channelId}>`
**权限**: `:approve`
**业务流程** (lock + 大事务):
1. lock `lock:org:channelApply:{applyId}`
2. 状态机 SM-3 跃迁 pending→approved
3. INSERT jst_channel: user_id=apply.user_id, channel_type=apply.channel_type, channel_name=apply.apply_name, contact_mobile=查 jst_user.mobile, status=1, auth_status=approved, auth_time=now()
4. INSERT sys_user_role: 给 apply.user_id 追加 jst_channel 角色 (从 sys_role 查 role_id where role_key='jst_channel')
5. UPDATE apply: status=approved, audit_user_id=current, audit_remark, audit_time=now()
6. **不要**修改 jst_user.user_type (用户可能同时是 student + channel,通过角色而非 user_type 区分)
7. 写 audit_log + 触发 mock 短信

### 6. (admin) POST /jst/organizer/channel/apply/{applyId}/reject
**入参**: `RejectReqDTO {@NotBlank String auditRemark}`
**业务**: 状态机 pending→rejected
**权限**: `:reject`

### 7. (admin) POST /jst/organizer/channel/apply/{applyId}/suspend
**用途**: 暂停已通过的渠道方
**权限**: `:suspend`
**业务**: 状态机 approved→suspended,UPDATE jst_channel.auth_status=suspended, status=0
