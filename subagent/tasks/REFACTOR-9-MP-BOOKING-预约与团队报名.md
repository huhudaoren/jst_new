# REFACTOR-9-MP-BOOKING — 预约与团队报名逻辑防呆

> 优先级：P1 | 预估：L | Agent：MiniProgram Agent
> 依赖：REFACTOR-1-DDL（预约时间段子表 + 团队字段）

---

## 一、背景

管理端已支持结构化预约时间段（jst_appointment_slot）和团队配置字段，小程序端需要严格对接新数据结构，实现防呆预约和团队报名流程。

## 二、预约改造

### 2.1 预约时间段选择（替换当前简化逻辑）

页面：`pages-sub/contest/appointment.vue`（个人预约）

**新流程**：
1. 进入预约页 → 获取赛事的 `appointmentSlotList`
2. 按日期分组展示（日期 Tab 或日历视图）
3. 每个时间段显示：时间范围 | 场地 | 剩余容量/总容量
4. 已满的时间段灰色不可选 + "已满"标记
5. 用户选择时间段 → 确认预约 → 调用预约接口（传 slotId）

**UI 设计**：
```
┌─ 4月15日 ─┬─ 4月16日 ─┬─ 4月17日 ─┐  ← 日期 Tab
│                                      │
│  09:00-10:00  会议室A  [剩余 8/20]   │  ← 可选，绿色
│  10:00-11:00  会议室A  [已满 20/20]  │  ← 不可选，灰色
│  14:00-15:00  会议室B  [剩余 15/20]  │  ← 可选
│  15:00-16:00  会议室B  [剩余 3/20]   │  ← 可选，橙色(快满)
│                                      │
│  [确认预约]                           │
└──────────────────────────────────────┘
```

### 2.2 预约确认弹窗

选择时间段后弹出确认面板：
- 显示：赛事名称、预约日期、时间、场地
- 提示剩余容量
- 如果 `allowRepeatAppointment == 0` 且已有预约 → 提示"您已预约过此赛事"
- 确认按钮 → 提交 → 成功动画 → 跳转我的预约

## 三、团队报名改造

### 3.1 团队报名入口

当赛事 `team_min_size > 0` 时，报名页增加"团队报名"入口（与个人报名并列）。

### 3.2 团队报名流程

页面：`pages-sub/contest/team-enroll.vue`（新增）

**Step 1 — 队长信息**：
- 根据 `team_leader_fields`（逗号分隔）渲染表单
- 如：姓名、手机、身份证、学校
- 复用 jst-dynamic-form 组件

**Step 2 — 添加队员**：
- 显示要求：最少 N 人，最多 M 人（来自 team_min_size/team_max_size）
- 队员列表（卡片形式）：姓名 | 手机 | 操作
- 「添加队员」按钮：
  - 方式 1：手动填写（弹出表单 popup）
  - 方式 2：微信分享邀请（生成邀请链接/小程序码）
  - 方式 3：从已有参赛档案选择
- 队员数量校验：
  - 少于 team_min_size → 按钮禁用 + 提示"还需添加 N 人"
  - 达到 team_max_size → 隐藏添加按钮 + 提示"已达最大人数"
- 可移除队员（队长不可移除）

**Step 3 — 确认提交**：
- 展示团队摘要：队长 + 所有队员 + 总费用
- 提交后进入支付流程（每个队员生成子订单）

### 3.3 微信邀请机制

- 队长创建团队后获得邀请码/链接
- `uni.share` 分享到微信好友/群
- 被邀请者点击 → 打开小程序 → 自动跳转团队页 → 填写自己信息 → 加入团队
- 实时显示已加入人数

### 3.4 防呆校验清单

- [ ] 团队人数 < min → 不允许提交
- [ ] 团队人数 > max → 不允许继续添加
- [ ] 队员手机号重复 → 提示
- [ ] 队长不可移除自己
- [ ] 预约时间段已满 → 不可选择
- [ ] 重复预约 → 提示（当 allowRepeatAppointment = 0）
- [ ] 已报名该赛事 → 提示（前端+后端双校验）
