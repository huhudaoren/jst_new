**对应原型**: `小程序原型图/my.html`
**视觉**: 顶部头像 + 昵称 + 等级 + 积分余额; 中间 9 宫格菜单; 底部退出登录
**菜单项**(本任务实现的菜单跳转):
- 我的资料 → /pages-sub/my/profile-edit
- 我的档案 → /pages-sub/my/participant

- 我的成绩/证书 → 占位

**双视角**: 本任务**仅做学生视角**,老师视角占位(toast "F6 完成后开放")

### 3. pages-sub/my/profile-edit.vue (新建)
**对应原型**: `小程序原型图/profile-edit.html`
**视觉**: 表单输入 (昵称/真实姓名/性别/生日/监护人姓名/监护人手机)
**数据**:
- onLoad 调 getProfile 拉取当前值预填
- 提交调 updateProfile (PUT)
- 提交后 toast 成功 + uni.navigateBack
**强约束**: 不允许显示/编辑 mobile 字段(只显示脱敏 mobileMasked)

### 4. pages-sub/my/participant.vue (新建)
**对应原型**: 自由设计(原型图无对应,可参考 my-binding.html 卡片样式)
**视觉**: 卡片列表
- 每张卡: 姓名 / 学校 / 班级 / 年龄 / 认领方式 / 认领时间
- 顶部按钮: 「+ 认领新档案」→ 弹出输入框输入 participantId → 调 claimParticipant
**数据**: onShow 调 myParticipants 拉取
**空状态**: 无档案时显示 "暂无已认领档案,通过老师/赛事方分享认领"

## 测试验证 (描述预期,不要求自动化)

### P1-1 登录流程
1. 启动 jst-uniapp (HBuilderX 内置浏览器)
2. 应自动跳到 login 页
3. 点 Mock 登录按钮 → 成功 → 跳 my 页
4. my 页应显示 "测试_张妈妈" 昵称

### P1-2 个人中心
1. 在 my 页应看到完整资料(头像兜底/昵称/手机号 138****0001/积分 0)
2. 点 9 宫格的 "我的资料" → 跳 profile-edit
3. profile-edit 应预填当前值
4. 改昵称为 "测试_张妈妈_新" → 提交 → toast "成功" → 返回 my → 看到新昵称

### P1-3 临时档案
1. 在 my 页点 "我的档案" → 跳 participant 页
2. 应看到 fixture 中已认领的 1 条 (3004 张小红)
3. 点 "+ 认领新档案" → 输入 3001 → 提交 → toast 成功 → 列表刷新看到 2 条 (3004 + 3001)
4. 再次点 "+ 认领新档案" → 输入 3001 → 应 toast 报错 (重复认领)

### P1-4 退出登录
1. 在 my 页点 "退出登录"
2. 应清空 token + 跳回 login 页

### P1-5 异常处理
1. 关闭后端 ruoyi-admin
2. 操作任意接口 → 应弹 "网络异常" toast
3. 启动后端再操作 → 恢复正常
