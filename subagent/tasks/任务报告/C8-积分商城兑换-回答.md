1. 按实测 schema 使用 `exchange_id` 作为主键，`order_id` 仅关联现金补差订单

3. `jst_points_account` 兼容现存 `owner_type in ('user','student')`，新补账户统一写 `user`
4. `jst_mall_goods.remark` 以 JSON 方式承载虚拟发放目标配置，避免杜撰不存在的列
