3. **WxChannelAuthApplyController base path 变更** — 从 `/jst/wx/channel/auth/apply` 改为 `/jst/wx/channel/auth`，原 `POST /` 变为 `POST /apply`，原 `GET /my` 变为 `GET /apply/my`。如有前端依赖旧路径需同步修改
4. **批量报名 (batch-enroll)** — 本期 MVP 仅做参赛人存在性校验，实际报名流程（表单提交+审核）由学生端完成。需后续对接 F9 EnrollRecordService
5. **auto-claim 端点** — 当前 mobile/name 参数为 null 传入 claimByAuto，实际需从 jst_user 查询当前用户手机号和姓名。需补充用户信息查询逻辑
6. **成绩/证书查询 SQL** — `selectStudentScores`/`selectStudentCerts` 查询的 `score_value`/`cert_url` 字段需确认是否已在 `jst_enroll_record` 表中存在，如不存在则返回空列表
7. **98-migration DDL 未执行** — 需在测试库执行验证（首跑+复跑幂等）

## F. 我做了任务卡之外的什么

1. **将 ChannelAuthApprovedEvent 从 jst-organizer 迁移到 jst-common** — 原因：jst-marketing（监听方）不依赖 jst-organizer，事件类放 jst-common 是正确的跨模块事件设计
2. **在 BizErrorCode 追加了 4 个错误码** — 任务卡明确要求
3. **在 RightsTemplateMapperExt 追加了 selectAutoGrantTemplates 方法** — Part D 监听器需要查询可自动发放的权益模板
4. **新建 ChannelSupplementService/ChannelSupplementLookupMapper** — Part C 的 4 个端点需要独立的查询/写入能力，原 ChannelDashboardService 职责不同
5. **在 ParticipantMapper 追加 insertParticipant** — Part E 批量创建需要
6. **WxParticipantController 改为无 @RequestMapping base path，使用方法级绝对路径** — 因为新端点 `/jst/wx/channel/participant/...` 与原 `/jst/wx/participant/...` 前缀不同
