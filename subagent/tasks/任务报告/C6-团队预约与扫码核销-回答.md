## F. 本次引发的错误与修正方案
- 启动报错：`org.apache.ibatis.type.TypeException: The alias 'JstTeamAppointmentMember' is already mapped to the value 'com.ruoyi.jst.order.domain.JstTeamAppointmentMember'`
- 根因：仓库 [application.yml](D:/coding/jst_v1/RuoYi-Vue/ruoyi-admin/src/main/resources/application.yml#L108) 配置了 `typeAliasesPackage: com.ruoyi.**.domain`，MyBatis 会按“简单类名”注册别名；本任务在 `jst-event` 新增了 `JstTeamAppointment`、`JstTeamAppointmentMember`、`JstAppointmentRecord`、`JstAppointmentWriteoffItem`，与 `jst-order` 现有同名实体冲突。
- 修改方案：不动 `jst-order` 既有实体，只把本次新增的 `jst-event` 侧 4 个实体重命名为 `EventTeamAppointment`、`EventTeamAppointmentMember`、`EventAppointmentRecord`、`EventAppointmentWriteoffItem`，并同步修改 Mapper 接口、ServiceImpl、Mapper XML、任务报告中的文件名引用。
- 修正后验证：已执行 `mvn -pl jst-event,jst-order -am compile -DskipTests`，结果 `BUILD SUCCESS`。

- 为了满足“jst-event 不能直接依赖 jst-order serviceImpl”，额外在 `jst-order` 中抽出了跨模块可注入的 `OrderCreationFacade`，把预约子订单创建封装成稳定接口。
