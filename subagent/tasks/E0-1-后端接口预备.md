## 交付物

### Part A — DDL 迁移脚本
**新建**：`架构设计/ddl/98-migration-phase-e-prep.sql`

幂等脚本，包含：
1. `ALTER TABLE jst_contest ADD COLUMN IF NOT EXISTS allow_appointment_refund TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否允许预约退款（Q-07 配置化）';`
2. `ALTER TABLE jst_channel_auth_apply ADD COLUMN IF NOT EXISTS reject_count INT NOT NULL DEFAULT 0 COMMENT '累计驳回次数（Q-02 限制 3 次）';`
3. `ALTER TABLE jst_channel_auth_apply ADD COLUMN IF NOT EXISTS locked_for_manual TINYINT(1) NOT NULL DEFAULT 0 COMMENT '>=3 次驳回后锁定';`
4. 新增 `sys_role` 两条：`jst_platform_auditor` / `jst_platform_finance`（走 `INSERT ... ON DUPLICATE KEY UPDATE` 幂等）
5. 测试库执行验证，首跑 + 复跑幂等

### Part B — 渠道认证申请后端接口（jst-organizer）
**修改**：`WxChannelAuthApplyController.java` + `ChannelAuthApplyServiceImpl.java`

新增/补齐接口：
- `GET /jst/wx/channel/auth/my` — 查询当前用户的认证申请记录（最新一条，含 `rejectCount/lockedForManual/reject_reason`）
- `POST /jst/wx/channel/auth/apply` — 提交申请（已有，需补齐 3 种 channelType 字段差异校验：teacher 必填 `teacher_cert_url`；organization 必填 `business_license_no` + `business_license_url`；individual 必填 `id_card_front_url`）
- `POST /jst/wx/channel/auth/resubmit/{id}` — 驳回后重提（服务端检查 `rejectCount < 3`，否则返回 `JST_CHANNEL_AUTH_LOCKED`）
- `POST /jst/wx/channel/auth/cancel/{id}` — 撤回申请（pending 状态）

服务层：
- 驳回时 `rejectCount++`，达到 3 时置 `lockedForManual=1`
- 审核通过时触发 `GrantInitialRightsEvent`（见 Part D）

### Part C — 渠道工作台补充接口（jst-channel）
**修改**：`WxChannelDashboardController.java`

新增/补齐：
- `GET /jst/wx/channel/rights/my` — 渠道方权益列表（从 `jst_user_rights` 按 `user_id` 查，返回含 `rightsName/remainQuota/expireTime`）
- `GET /jst/wx/channel/students/{studentId}/score` — 查指定绑定学生的成绩列表（校验绑定关系 + 赛事已发布成绩）
- `GET /jst/wx/channel/students/{studentId}/cert` — 查指定绑定学生的证书列表
- `POST /jst/wx/channel/binding/{bindingId}/unbind` — 渠道方**自助解绑**（Q-01，走事务 + 记录操作日志；不回溯历史返点）

### Part D — 权益自动发放（jst-points）
**修改**：`RightsUserServiceImpl.java` + 新建 `GrantInitialRightsListener.java`

- 监听 `ChannelAuthApprovedEvent`（Part B 发事件）
- 查询该用户 `user_level`（无则创建为 level_1）
- 按 `jst_level_rights_config` 自动发放对应权益到 `jst_user_rights`
- 事务：`@Transactional(rollbackFor=Exception.class)`

### Part E — 批量报名与临时参赛档案后端（jst-user）
**修改/新建**：`Participant.java` 域 + `WxParticipantController.java`

- `POST /jst/wx/channel/participant/batch-create` — 渠道方批量创建临时档案（接收 JSON 数组或 Excel 解析后的 List）
- `POST /jst/wx/channel/participant/batch-enroll` — 批量报名（接收 `{contestId, participantIds, userIds}`，内部循环走 F9 报名创建逻辑）
- `POST /jst/wx/participant/auto-claim` — 学生登录时自动触发：按"监护人手机号 + 参赛人姓名"精确匹配临时档案，匹配到则写入 `participant_user_map` 并合并订单归属

### Part F — 预约退款配置读取（jst-event）
**修改**：`AppointmentServiceImpl.cancelIndividual / cancelTeam`

- 取消预约时先读 `jst_contest.allow_appointment_refund`
- `=0`：仅允许取消预约状态，**不触发退款**，返回提示"本活动不支持退款"
- `=1`：正常走退款链路（复用 C4 范式）
- admin 特批退款走后台（E-4-WEB 批次）

### Part G — 错误码补齐
**修改**：`BizErrorCode.java`

新增：
- `JST_CHANNEL_AUTH_LOCKED`（10040）— "认证申请已达最大驳回次数，请联系客服"
- `JST_APPOINTMENT_REFUND_NOT_ALLOWED`（10041）— "本活动不支持预约退款"
- `JST_CHANNEL_UNBIND_FAILED`（10042）— "解绑学生失败"
- `JST_PARTICIPANT_AUTO_CLAIM_CONFLICT`（10043）— "临时档案自动认领存在多个候选，需人工处理"
