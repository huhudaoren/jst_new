## 接口

### POST /jst/wx/appointment/apply ⭐ 创建个人预约
- 入参：`{ contestId, sessionCode, appointmentDate, participantId?, couponId?, pointsToUse? }`
- 逻辑：
  1. 登录用户
  2. 校验 contest 支持预约、日期合法
  3. 若 `allow_repeat_appointment=0`，查是否已有 booked 记录 → 拒绝
  4. `lock:appointment:book:{contestId}:{sessionCode}`
  5. 校验名额（SUM 个人 + team total - cancelled）
  6. 若 price > 0：调 `OrderCreationFacade.createForAppointment(dto)` 拿 orderId（复用 C6 暴露的接口；订单 pay 逻辑由学生后续走 C2 的 /pay/mock-success 完成）
  7. INSERT `jst_appointment_record` (user_id, main_status=`booked`, order_id?, qr_code)
  8. 按 contest.writeoff_config 生成 N 条 writeoff_item
  9. 返回 `AppointmentApplyResVO { appointmentId, orderId?, qrCodes[] }`

### POST /jst/wx/appointment/{id}/cancel
- 校验 ownership + status=`booked` 且无 used 子项
- `lock:appointment:cancel:{id}`
- `UPDATE record SET main_status='cancelled'`
- 批量 `UPDATE writeoff_item SET status='voided' WHERE appointment_id=? AND status='unused'`
- 若 order_id 存在：调 RefundService 发起 admin 自动退款（复用 C4）或本期仅标 order cancelled 留 admin 手动处理（**决策：本期仅标 order.cancelled，提示用户联系客服退款**，避免 C7 与 C4 过度耦合）

### GET /jst/wx/appointment/contest/{contestId}/remaining
- 返回指定场次剩余名额，供前端展示

### GET /jst/wx/appointment/my
- 已由 C6 实现，本任务扩展加"是否个人/团队"字段

### GET /jst/wx/appointment/{id}
- 已由 C6 实现，本任务验证个人路径字段齐全
