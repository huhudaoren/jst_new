# 任务卡 F4 - 学生-渠道方绑定

## 阶段 / 模块
阶段 B / jst-user

## 业务背景
学生(或家长)在小程序内可绑定到一个老师/机构(渠道方),后续报名/购买课程会归属到该渠道,渠道方据此拿返点。
- 同一学生同时仅 1 条 active 绑定
- 切换绑定 = 旧记录置 replaced + 新记录置 active
- 解绑 = 当前 active 置 manual_unbound

## 涉及表
- jst_student_channel_binding (主)
- jst_channel (校验渠道存在 + 取名称用)
- jst_user (校验当前用户)

## 涉及状态机
**SM-15** `jst_student_channel_binding.status`:
```
active → (expired | replaced | manual_unbound)
```
- 跃迁规则: active 可去任意终态;终态不可再改
- 业务约束: 同一 user_id 同时仅 1 条 active

## 涉及锁
- `lock:bind:{userId}` (wait 3s, lease 5s) - 切换/解绑都加

## 涉及权限标识
- 小程序端: `hasRole('jst_student')` (角色级)
- admin 端: `jst:user:binding:list`, `jst:user:binding:forceUnbind`

## 接口契约

### 1. GET /jst/wx/user/binding
**用途**: 我的绑定列表(通常 1 条 active + 多条历史)
**入参**: 无 (userId 从 token 取)
**出参**: `AjaxResult<List<BindingVO>>`
```java
class BindingVO {
    Long bindingId;
    Long channelId;
    String channelName;       // 冗余,从 jst_channel 关联取
    String channelType;       // teacher/organization/individual
    Date bindTime;
    Date unbindTime;          // active 时为 null
    String status;            // active/expired/replaced/manual_unbound
}
```
**权限**: `hasRole('jst_student')`
**业务**: 直接 SELECT WHERE user_id = currentUser ORDER BY bind_time DESC

### 2. POST /jst/wx/user/binding/switch ⭐ 核心
**用途**: 切换/新建绑定
**入参**:
```java
class SwitchBindingReqDTO {
    @NotNull Long channelId;  // 目标渠道方ID
}
```
**出参**: `AjaxResult<BindingVO>` (新建的 active 绑定)
**权限**: `hasRole('jst_student')`
**业务流程** (必须在 lock + tx 内):
1. lock `lock:bind:{userId}` (wait 3, lease 5)
2. 校验 jst_channel.channel_id 存在且 status=1 且 auth_status=approved
3. 查当前 active 绑定:
   - 已绑同一 channelId → 抛业务异常 `BizErrorCode.JST_USER_BIND_CONFLICT` (新增错误码)
   - 已绑别的 → 状态机校验 active→replaced,UPDATE 旧记录 status=replaced, unbind_time=now()
4. INSERT 新记录: status=active, bind_time=now()
5. 同时 UPDATE jst_user.bound_channel_id = newChannelId
6. 返回 BindingVO

### 3. POST /jst/wx/user/binding/unbind
**用途**: 用户主动解绑
**入参**:
```java
class UnbindReqDTO {
    @Size(max=255) String reason;
}
```
**出参**: `AjaxResult<Void>`
**权限**: `hasRole('jst_student')`
**业务**:
1. lock `lock:bind:{userId}`
2. 查 active 绑定,无则抛"无绑定可解除"
3. 状态机 active→manual_unbound,UPDATE status + unbind_time + unbind_reason + operator_id=currentUser
4. UPDATE jst_user.bound_channel_id = NULL

### 4. (admin) GET /jst/user/binding/list
**用途**: admin 查询绑定列表(支持按 userId/channelId/status 过滤)
**入参**: `BindingQueryReqDTO {Long userId, Long channelId, String status}`
**出参**: `TableDataInfo<BindingVO>`
**权限**: `@PreAuthorize("@ss.hasPermi('jst:user:binding:list')")`
**业务**: startPage + selectList,默认按 bind_time DESC

### 5. (admin) POST /jst/user/binding/{bindingId}/force-unbind
**用途**: 平台强制解绑(争议处理)
**入参**: `@PathVariable Long bindingId, @RequestParam String reason`
**出参**: `AjaxResult<Void>`
**权限**: `@PreAuthorize("@ss.hasPermi('jst:user:binding:forceUnbind')")`
**业务**: lock + 状态机校验 + UPDATE,reason 必填
