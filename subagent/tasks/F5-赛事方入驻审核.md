# 任务卡 F5 - 赛事方入驻申请 + 审核

## 阶段 / 模块
阶段 B / jst-organizer (主) + 改 jst-event 的 Service (次)

## 业务背景
赛事方(机构/培训机构)通过公开链接提交入驻申请,平台运营在管理后台审核;通过后系统:
1. 创建 jst_event_partner 主档
2. 创建 sys_user 账号 (用于 PC 端登录工作台)
3. 把 sys_user.user_id 回填到 jst_event_partner.user_id (DDL 已就位)
4. 给该 sys_user 赋角色 jst_partner
5. 短信/邮件通知申请人审核结果

## 涉及表
- jst_event_partner_apply (主)
- jst_event_partner (审核通过后创建)
- sys_user (审核通过后创建若依用户)
- sys_user_role (审核通过后赋 jst_partner 角色)
- sys_role (查 role_id where role_key='jst_partner')

## 涉及状态机
**SM-4** `jst_event_partner_apply.apply_status`:
```
draft → pending → (approved | rejected | supplement)
supplement → pending  (重提)
```

## 涉及锁
- `lock:org:apply:{applyId}` (wait 3s, lease 10s) - 审核操作

## 涉及权限标识
- 公开提交申请: `@Anonymous` (无需登录)
- 申请人查询自己的申请: `hasRole('jst_student')` (临时用学生角色,后续 F2 完成手机号绑定后可改用 mobile 反查)
- admin: `jst:organizer:partnerApply:list`, `:detail`, `:approve`, `:reject`, `:supplement`

## 接口契约

### 1. POST /jst/public/organizer/partner/apply  ⭐ 公开
**用途**: 公开页提交入驻申请
**入参**:
```java
class PartnerApplyReqDTO {
    @NotBlank @Size(max=128) String partnerName;
    @NotBlank @Size(max=64) String contactName;
    @NotBlank @Pattern(regexp="^1[3-9]\\d{9}$") String contactMobile;
    @Size(max=64) String businessLicenseNo;
    String qualificationJson;     // 资质材料JSON
    String settlementInfoJson;    // 结算信息JSON
    String invoiceInfoJson;       // 发票信息JSON
    String contractFilesJson;     // 合同附件JSON
}
```
**出参**: `AjaxResult<{applyId, applyNo}>`
**权限**: `@Anonymous`
**业务**:
1. 校验同一 contact_mobile 是否已有 status in (pending, approved) 的申请 → 拒绝重复
2. INSERT 状态 = pending,生成 apply_no = "PA" + 雪花ID
3. 写 audit_log
4. 发短信通知"申请已提交"(本期 mock 不实发,只 log)

### 2. GET /jst/public/organizer/partner/apply/{applyNo}/status
**用途**: 公开查询申请状态(凭单号)
**入参**: `@PathVariable String applyNo`
**出参**: `AjaxResult<{applyStatus, auditRemark, submitTime, auditTime}>`
**权限**: `@Anonymous`

### 3. (admin) GET /jst/organizer/partner/apply/list
**用途**: 审核员查询待审列表
**入参**: `PartnerApplyQueryDTO {String applyStatus, String partnerName, Date submitTimeStart, ...}`
**出参**: `TableDataInfo<PartnerApplyListVO>`
**权限**: `@PreAuthorize("@ss.hasPermi('jst:organizer:partnerApply:list')")`

### 4. (admin) GET /jst/organizer/partner/apply/{applyId}
**用途**: 审核详情
**出参**: `AjaxResult<PartnerApplyDetailVO>` (含全部 JSON 字段反序列化)
**权限**: `:detail`

### 5. (admin) POST /jst/organizer/partner/apply/{applyId}/approve  ⭐ 核心
**用途**: 通过审核
**入参**:
```java
class ApproveReqDTO {
    String auditRemark;
    @NotBlank String username;        // 给赛事方分配的用户名
    String email;                     // 邮件
}
```
**出参**: `AjaxResult<{partnerId, userId}>`
**权限**: `:approve`
**业务流程** (lock + 大事务):
1. lock `lock:org:apply:{applyId}`
2. 状态机 SM-4 跃迁 pending→approved (校验 supplement→pending→approved 也合法)
3. INSERT jst_event_partner: 字段从 apply 全量复制,audit_status=approved,coop_status=active
4. 创建 sys_user: username=入参,密码=随机12位(BCrypt 加密),user_type=01(企业类型),phonenumber=apply.contact_mobile
5. INSERT sys_user_role: 关联 jst_partner 角色 (从 sys_role 查 role_id where role_key='jst_partner')
6. UPDATE jst_event_partner SET user_id=新建的 sys_user.user_id WHERE partner_id=刚创建的
7. UPDATE apply: status=approved, audit_remark, audit_user_id=current, audit_time=now()
8. 写 audit_log + 触发 mock 短信日志("审核通过,用户名 xxx 初始密码 yyy")
9. 返回 partnerId + userId

### 6. (admin) POST /jst/organizer/partner/apply/{applyId}/reject
**用途**: 驳回
**入参**: `RejectReqDTO {@NotBlank @Size(max=255) String auditRemark}`
**业务**: 状态机 pending→rejected,UPDATE remark + 触发 mock 短信
**权限**: `:reject`

### 7. (admin) POST /jst/organizer/partner/apply/{applyId}/supplement
**用途**: 要求补材料
**入参**: `SupplementReqDTO {@NotBlank String supplementRemark}`
**业务**: 状态机 pending→supplement
**权限**: `:supplement`
