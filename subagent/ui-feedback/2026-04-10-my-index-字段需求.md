# UI 字段需求 - 个人中心 (pages/my/index)

> **提交者**：UI Polish Agent（专职视觉优化，不改业务逻辑/后端代码）
> **职责**：对照原型图调整 `jst-uniapp` 小程序界面样式、结构层次、Design Token 对齐。不修改 JS/API/Store/后端。当发现后端 VO 字段不足或页面路由缺失时，通过本反馈文档通知主 Agent 安排处理。
> **工作上下文**：用户提供了 `差异文件/my界面-双视角-*.png`（当前实现截图）与 `差异文件/my界面-双视角-*原型.png`（目标原型），要求按原型重写个人中心界面。

## 场景
为实现原型图 `差异文件/my界面-双视角-学生视角原型.png` 中的完整学生/渠道双视角布局，
当前后端 profile VO 字段不足，且部分页面路由不存在。

## 接口
- 路径：`GET /jst/wx/user/profile` (由 store/user.js fetchProfile 调用)
- 当前 VO 字段：nickname / avatar / mobileMasked / currentLevelId / availablePoints / totalPoints / growthValue / userType

## 需要补的字段

| 字段名 | 类型 | 说明 | 视觉用途 | 必填 |
|---|---|---|---|---|
| enrollCount | Integer | 报名记录总数 | 顶部 summary 卡第 1 格 | 是 |
| scoreCount | Integer | 成绩记录总数 | 顶部 summary 卡第 2 格 | 是 |
| certCount | Integer | 证书总数 | 顶部 summary 卡第 3 格 | 是 |
| courseCount | Integer | 在学课程数 | 顶部 summary 卡第 4 格 | 是 |
| frozenPoints | Integer | 冻结积分 | 等级卡中间数字 | 否 |
| levelName | String | 等级名称（如"白银学员"/"黄金渠道"）| 等级卡左上标题 | 否 |
| couponCount | Integer | 可用优惠券数 | 权益区"我的优惠券"红点 | 否 |
| unreadMsgCount | Integer | 未读消息数 | 消息区"我的消息"红点 | 否 |
| channelBinding | Object | 绑定的渠道方信息 { name, school, remark } | 绑定卡展示 | 否 |

## 影响范围
- 前端：`pages/my/index.vue` (template 已按原型重写，缺失字段暂显 0 或不渲染)
- 后端：UserProfileVO 补字段 + 对应 SQL count 查询

## 兼容方案
- enrollCount/scoreCount/certCount/courseCount 缺失 → 显示 0
- frozenPoints 缺失 → 不显示冻结积分行
- levelName 缺失 → 显示 "Lv.X"
- couponCount/unreadMsgCount 缺失 → 不显示红点
- channelBinding 缺失 → 不渲染绑定卡
