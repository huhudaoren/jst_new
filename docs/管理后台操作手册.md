## 一、系统概述

### 1.1 系统定位

竞赛通管理后台是平台运营人员、赛事方、渠道管理员的统一工作入口。采用 Vue 2 + Element UI 构建，支持 PC 浏览器和手机浏览器双端访问。

### 1.2 核心功能模块

| 模块 | 功能 | 典型用户 |
|---|---|---|
| 运营看板 | KPI 总览、待办事项、快速操作 | 运营人员 |
| 赛事管理 | 赛事审核、上下线、编辑 | 运营人员 |
| 报名管理 | 报名审核、补充材料通知 | 运营人员 |
| 课程管理 | 课程创建、编辑、上下线 | 内容运营 |
| 订单/退款 | 订单查询、退款审核、特批 | 客服/财务 |
| 渠道管理 | 渠道认证审核、绑定管理、提现审核 | 渠道运营 |
| 积分/营销 | 积分规则、优惠券发放、权益管理 | 营销运营 |
| 财务管理 | 合同、发票、打款记录 | 财务人员 |
| 风控管理 | 风控规则、告警、黑名单、审计日志 | 安全运营 |
| 赛事方工作台 | 赛事创建、报名审核、成绩证书、结算 | 赛事方 |

### 1.3 页面通用操作

所有管理页面遵循统一交互规范：

- **Hero 区域**：页面顶部展示模块名称、功能说明、帮助按钮（`?` 图标）
- **搜索面板**：支持关键字、状态下拉、时间范围等筛选条件
- **数据表格**：PC 端表格视图，手机端自动切换为卡片列表
- **详情抽屉**：点击「详情」按钮，右侧弹出详情面板
- **操作权限**：所有操作按钮受权限控制，无权限时按钮不显示
- **帮助提示**：Hero 区右上角 `?` 按钮，点击查看操作说明

## 二、登录与角色

### 2.1 登录

1. 打开管理后台地址
2. 输入用户名和密码
3. 输入验证码
4. 点击「登录」

### 2.2 角色说明

| 角色 | 权限范围 | 说明 |
|---|---|---|
| **超级管理员** (admin) | 全部功能 | 系统最高权限 |
| **平台运营** (platform_op) | 赛事/报名/订单/渠道/营销/公告 | 日常运营 |
| **学生** (student) | 仅小程序端 | 无后台权限 |
| **渠道方** (jst_channel) | 渠道工作台 | 返点、提现、学生管理 |
| **赛事方** (jst_partner) | 赛事方工作台 | 赛事管理、报名审核、成绩证书 |

### 2.3 菜单结构

登录后左侧菜单根据角色动态加载。主要一级菜单：

```
├── 数据看板
│   └── 运营数据
├── 渠道管理
│   ├── 渠道认证管理
│   ├── 渠道列表
│   ├── 绑定管理
│   └── 预约管理
├── 赛事方工作台（赛事方角色专属）
│   ├── 工作台首页
│   ├── 赛事管理
│   ├── 报名审核
│   ├── 成绩管理
│   ├── 证书管理
│   ├── 结算中心
│   ├── 现场核销
│   └── 账号设置
└── 若依系统管理（管理员专属）
    ├── 用户管理
    ├── 角色管理
    ├── 菜单管理
    └── ...
```

> 注：业务管理页面（赛事/课程/订单/积分/营销等）的菜单由代码生成器自动注册，可在「菜单管理」中调整顺序和可见性。

## 四、赛事管理

### 4.1 赛事列表

路径：赛事管理 → 赛事列表

**搜索条件**：赛事名称、分类、审核状态、上线状态、赛事方名称

**操作**：

| 操作 | 说明 | 权限 |
|---|---|---|
| 审核通过 | 通过赛事方提交的赛事 | jst:event:contest:audit |
| 审核驳回 | 驳回赛事并填写原因 | jst:event:contest:audit |
| 上线 | 将审核通过的赛事发布到前端 | jst:event:contest:online |
| 下线 | 将已上线赛事下线 | jst:event:contest:offline |
| 详情 | 查看赛事完整信息 | jst:event:contest:query |
| 编辑 | 修改赛事信息 | jst:event:contest:edit |

### 4.2 赛事编辑

赛事编辑页采用 Tab 分区，共 6 个模块：

**Tab A — 基本信息**
- 赛事名称、分类、组别
- 封面图上传
- 报名价格
- 赛事详情（富文本编辑器）

**Tab B — 时间与场次**
- 报名开始/截止时间
- 赛事开始/结束时间
- 赛程安排（可视化表格：添加/删除场次，每行含名称、时间、地点、说明）

**Tab C — 奖项与标签**
- 奖项设置（可视化表格：奖项名称、等级、说明）
- 常见问题 FAQ（可视化表格：问题、回答）
- 推荐标签（标签输入，动态添加/删除）

**Tab D — 成绩与证书**
- 成绩规则配置：
  - 发布模式：手动发布 / 自动发布（下拉选择）
  - 成绩项：动态标签输入（如"总分"、"单项1"）
- 证书规则配置：
  - 颁发模式：手动颁发 / 自动颁发（下拉选择）
  - 证书模板：从已有模板中选择

**Tab E — 预约与核销**
- 是否支持预约（开关）
- 预约容量、重复预约次数
- 核销配置：
  - 核销方式：二维码 / 手动输入（下拉选择）
  - 是否需要签到（开关）

**Tab F — 退款配置**
- 是否允许退款（开关）
- 售后期限（天数）

### 4.3 赛事方入驻管理

路径：赛事方入驻管理

- 查看赛事方的入驻申请
- 审核通过/驳回
- 查看申请详情（资质材料、联系方式）

## 六、课程管理

### 6.1 课程列表

路径：课程管理 → 课程列表

**搜索条件**：课程名称、类型（普通/AI）、状态

**操作**：新建课程、编辑、上线、下线、详情

### 6.2 课程编辑

课程编辑页采用 5 个 Tab：

**Tab A — 基本信息**
- 课程名称、类型（普通/AI）
- 封面图上传
- 简介
- 现金价格、积分价格
- 可见范围（全部/仅学生/仅渠道）

**Tab B — 讲师信息**
- 讲师姓名
- 讲师头像上传
- 讲师简介

**Tab C — 课程目录**

这是课程编辑的核心功能：

1. 点击「添加章节」创建新章节
2. 输入章节名称
3. 在章节下点击「添加课时」
4. 每个课时可编辑：课时名称、时长、是否免费
5. 支持上下移动排序章节和课时
6. 支持删除章节和课时

```
第一章：入门基础
├── 1.1 课程介绍（05:30）[免费]
├── 1.2 环境准备（12:00）
└── 1.3 第一个项目（15:30）
第二章：进阶技巧
├── 2.1 高级特性（20:00）
└── 2.2 实战演练（25:00）
```

**Tab D — 课程介绍**
- 富文本编辑器，支持图文混排

**Tab E — 统计信息**
- 课时数（可编辑）
- 总时长（如"12小时30分"）
- 学习人数（只读，由系统统计）

## 七、订单管理

路径：订单管理

### 7.1 订单列表

**搜索条件**：订单号、用户名、赛事名、订单状态、支付方式、创建时间范围

### 7.2 订单状态流转

```
创建(pending_pay) → 已支付(paid) → 服务中(in_service) → 已完成(completed)
                  ↘ 超时取消(cancelled)
                  ↘ 用户取消(cancelled)
```

### 7.3 订单详情

点击「详情」查看：
- 基本信息：订单号、创建时间、状态
- 用户信息：用户名、手机号
- 商品信息：赛事/课程名称、参赛人
- 金额明细：
  - 标价金额
  - 优惠券抵扣
  - 积分抵扣
  - 实付金额
- 支付信息：支付方式、支付时间、交易号
- 退款信息（如有）

### 7.4 订单明细行

路径：订单明细行管理

每笔订单可能包含多个明细行（赛事报名、课程购买等）。

- 查看各明细行的分摊金额
- 金额展示：标价、优惠券分摊、积分分摊、净实付、服务费、渠道返点
- 点击订单号可跳转到订单详情

## 八、退款管理

路径：退款管理

### 8.1 退款列表

**搜索条件**：退款单号、订单号、用户名、退款状态、申请时间范围

### 8.2 退款审核

| 操作 | 说明 | 权限 |
|---|---|---|
| 审核通过 | 同意退款，系统自动执行退款 | jst:order:refund:audit |
| 审核驳回 | 拒绝退款，填写原因 | jst:order:refund:audit |
| 平台特批 | 赛事方驳回后，平台可特批退款 | jst:order:refund:override |
| 详情 | 查看退款详情 | jst:order:refund:query |

### 8.3 退款规则

- 本期仅支持全额退款
- 退款顺序：微信现金 → 积分 → 优惠券
- 退款成功后：
  - 订单状态变为 cancelled
  - 报名状态变为 cancelled
  - 渠道返点回退（未提现直接回退，已提现生成负向台账）
- 优惠券回退条件：原券在有效期内

## 二十三、常见问题

### Q1：登录后看不到某些菜单？
A：菜单根据角色权限动态加载。联系管理员确认你的角色是否分配了对应菜单权限。

### Q2：赛事审核通过后还能修改吗？
A：审核通过后需要先下线，修改后重新提审。已有报名的赛事修改需谨慎。

### Q3：退款审核通过后资金多久到账？
A：系统自动执行退款。微信支付部分 1-3 个工作日到账，积分和优惠券即时回退。

### Q4：如何导出数据？
A：大部分列表页面支持「导出」按钮，导出为 Excel 文件。导出按钮需要对应的导出权限。

### Q5：渠道提现审核后如何打款？
A：审核通过后，财务人员在银行完成转账，然后回到系统点击「确认打款」并上传凭证。

### Q6：如何隐藏不需要的菜单？
A：管理员进入 系统管理 → 菜单管理，找到对应菜单将「显示状态」改为「隐藏」。

### Q7：手机端如何访问管理后台？
A：直接在手机浏览器输入后台地址即可。所有页面已做响应式适配，手机端会自动切换为卡片布局。

### Q8：赛事方如何入驻？
A：通过小程序「公开页面 → 赛事方入驻」提交申请，或由管理员在后台「赛事方入驻管理」中手动创建。
