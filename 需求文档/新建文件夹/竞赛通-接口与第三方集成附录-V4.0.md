# 竞赛通-接口与第三方集成附录-V4.0

## 1. 文档定位

- 本附录定义 V4.0 对外开发交接所需的接口分层、响应规范、第三方集成约束和回调规则。
- 实际网关、域名、鉴权中间件可在实施阶段落地，但路径语义、关键字段、幂等规则、回调口径需保持一致。
- 本附录与《系统架构设计文档》《业务状态机与字典附录》共同构成接口单一事实源；接口、字段、状态值如有冲突，以三者统一后的修订版本为准，不允许代码先行漂移。

## 2. 接口总规范

### 2.1 接口分层

| 前缀 | 说明 | 使用端 |
|------|------|--------|
| `/api/app` | 学生端、渠道方微信小程序业务接口 | 微信小程序 |
| `/api/admin` | 平台管理 Web 与平台 H5 审核端接口 | 平台管理 Web、平台 H5 审核端 |
| `/api/partner` | 赛事方工作台接口 | 赛事方 Web |
| `/api/open` | 第三方回调、开放接入接口 | 支付、AI、物流、签章、开票等 |

### 2.2 通用响应体

```json
{
  "code": 200,
  "msg": "success",
  "traceId": "20260331-xxxx",
  "data": {}
}
```

### 2.3 通用约束

1. 所有写接口必须支持幂等，默认使用 `bizNo` 或 `X-Request-Id` 作为幂等键。
2. 所有回调接口需校验签名、时间戳、随机串，并记录原始报文。
3. 所有金额字段统一为元，保留两位小数；比例字段使用小数，不使用百分号。
4. 所有分页接口统一支持 `pageNum`、`pageSize`、`orderByColumn`、`isAsc`。
5. 所有导出接口走异步导出任务，生成文件后发送站内消息。
6. 平台管理 Web 与平台 H5 审核端共用同一套管理接口，仅通过角色权限和 `clientType` 控制是否可执行审核和打款动作。
7. `/api/app/enroll/create`、`/api/app/enroll/submit` 默认使用请求头 `X-Request-Id` 承载幂等键 `enrollDraftNo`、`submitNo`。
8. 微信小程序业务端登录态需绑定 `openid/unionid`，并与业务账号、手机号建立映射关系。
9. 平台管理 Web 与平台 H5 审核端必须分别发放 `clientType=admin_web/h5_audit` 会话，允许共享人员身份，不允许共享会话上下文。

### 2.5 协作与接口冻结约束

1. 新增或修改接口前，必须先冻结 `Path / Method / DTO / 状态值 / 权限标识 / 幂等键`，再进入前后端并行开发。
2. 接口字段名、状态值、金额口径、分页参数以本附录为准，前端不得自行发明字段或重命名后再映射。
3. 涉及破坏性变更时，必须提供兼容期方案、版本迁移方案或灰度切换方案，不允许直接覆盖旧行为。
4. 涉及 `通用 DTO / 公共枚举 / 公共字典 / 登录态 / 文件上传 / 回调验签` 的改动，提交前必须同步相关负责人并更新附录。
5. 管理员高频移动操作默认优先落到 `平台 H5 审核端`，复杂配置、批量导入导出、深度报表分析保留在 `平台管理 Web`。
6. 所有联调、验收、问题排查必须可追踪 `traceId`；涉及审核、打款、核销、红冲、冻结的接口必须落审计日志。

### 2.4 回调与重试规则

| 项 | 规则 |
|----|------|
| 回调成功响应 | 返回 `{"code":0,"msg":"ok"}` |
| 回调超时 | 5 秒超时，第三方按供应商规则重试 |
| 我方重试 | 异步任务默认最多重试 5 次，指数退避 |
| 签名校验失败 | 直接拒绝并记录安全日志 |
| 幂等冲突 | 返回成功态，但不重复处理业务 |

#### 3.1.1 `/api/admin/auth/login`
请求字段：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `username` | `string` | 是 | 管理员用户名 |
| `password` | `string` | 是 | 管理员密码 |
| `captcha` | `string` | 否 | 图形验证码，开启验证码时必填 |
| `uuid` | `string` | 否 | 图形验证码会话标识，开启验证码时必填 |
| `clientType` | `string` | 否 | 管理端终端类型，允许值 `admin_web/h5_audit`；未传时默认 `admin_web` |

响应字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| `token` | `string` | 当前终端专属会话 token |
| `clientType` | `string` | 本次登录签发的终端类型，返回值仅允许 `admin_web/h5_audit` |
| `user` | `object` | 当前管理员档案，沿用若依 `SysUser` 输出口径 |
| `roles` | `array<string>` | 当前管理员角色标识集合 |
| `permissions` | `array<string>` | 当前管理员权限标识集合 |

行为说明：
1. 平台管理 Web 必须使用 `clientType=admin_web`，平台 H5 审核端必须使用 `clientType=h5_audit`。
2. `admin_web` 与 `h5_audit` 允许共享同一管理员身份，但必须签发不同登录会话，不允许跨端共用 token。
3. `clientType` 与接口前缀共同构成会话隔离边界；管理端 token 不得访问 `/api/app/**`，小程序 token 不得访问 `/api/admin/**`。

#### 3.1.2 `/api/admin/auth/info`
请求参数：无；该接口要求管理端登录态。

响应字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| `clientType` | `string` | 当前登录会话终端类型，返回值仅允许 `admin_web/h5_audit` |
| `user` | `object` | 当前管理员档案，沿用若依 `SysUser` 输出口径 |
| `roles` | `array<string>` | 当前管理员角色标识集合 |
| `permissions` | `array<string>` | 当前管理员权限标识集合 |

行为说明：
1. 平台管理 Web 与平台 H5 审核端均走该接口回拉当前登录态，但前端必须根据 `clientType` 控制当前终端可展示的操作入口。
2. 当前会话 `clientType` 与登录签发值保持一致，不允许前端本地改写后继续使用。

#### 3.1.3 `/api/app/bind/current`
请求参数：该接口要求 app 登录态。

请求字段：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `studentId` | `long` | 否 | 学生档案 ID；未传时默认查询当前登录用户主学生档案的有效绑定 |

响应字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| `bindId` | `long` | 当前有效绑定记录 ID |
| `channelId` | `long` | 当前绑定渠道方 ID |
| `channelName` | `string` | 当前绑定渠道方名称 |
| `status` | `string` | 当前绑定状态，当前最小正式值为 `active` |

行为说明：
1. 若当前登录用户没有学生档案，或当前学生没有有效绑定关系，接口返回 `data=null`，前端展示真实空态。
2. 若传入 `studentId`，服务端必须校验该学生档案归属于当前登录用户，不允许跨用户查询他人绑定关系。
3. 当前页面只承接“查看当前关系”能力；扫码绑定、绑定码录入、关系变更提交继续按后续冻结接口补齐，不允许前端自行发明协议。

## 5. 异常与补偿策略

| 场景 | 失败后处理 |
|------|------------|
| 微信支付成功但我方未落单 | 依赖支付回调补单，并生成支付对账任务 |
| 订单超时未支付 | 由 `jst_async_task(close_order)` 定时关闭订单，并释放锁定中的优惠券与积分 |
| 退款成功但返点已打款 | 生成负向返点台账，后续提现自动抵扣 |
| 合同签署回调失败 | 定时轮询签署平台补单 |
| 开票平台返回失败 | 发票申请回退至 `reviewing`，财务重新处理 |
| 物流回调失败 | 保留人工录入物流能力，允许后台手工补录 |
| OpenMAIC 回调失败 | 写入回调重试表，最多重试 5 次 |

| `payTime` | string | 支付时间，可空 |
| `expireTime` | string | 支付截止时间，可空 |
| `closeTime` | string | 关闭时间，可空 |
| `closeReason` | string | 关闭原因，可空 |
| `transferVoucherStatus` | string | 转账凭证状态：`none/pending/approved/rejected` |
| `transferVoucherStatusLabel` | string | 转账凭证状态文案 |
| `canPay` | boolean | 是否可继续支付 |
| `canCancel` | boolean | 是否可取消订单 |
| `canViewDetail` | boolean | 是否可查看详情 |
| `canReapply` | boolean | 是否可重新报名，仅赛事订单可能为 `true` |
| `canContinueLearn` | boolean | 是否可继续学习，仅课程订单可能为 `true` |

行为约束：

- 当前接口仅返回当前登录用户自己的订单，不返回渠道工作台订单。
- `summary` 统计受 `bizType` 影响，不受 `payStatus` 影响。
- 赛事订单的 `displayStatus` 会叠加报名审核状态；课程订单以支付状态为主。
- 前端禁止用本地假数据回退，只允许做 `loading/empty/error` 三态处理。

### 7.2 POST /api/app/order/cancel/{orderNo}

用途：当前登录用户取消未支付或转账驳回订单。

路径参数：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `orderNo` | string | 是 | 订单号 |

响应字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `orderNo` | string | 订单号 |
| `payStatus` | string | 取消后的支付状态，固定返回 `closed` |
| `enrollStatus` | string | 赛事报名单状态；课程订单为空 |
| `closeTime` | datetime | 关闭时间 |
| `closeReason` | string | 关闭原因 |

行为约束：

- 仅允许取消 `pending_pay/rejected` 订单。
- 取消订单后需要释放锁定中的优惠券和积分。
- 赛事订单取消后，报名单状态写为 `cancelled`；课程订单无报名单状态联动。
