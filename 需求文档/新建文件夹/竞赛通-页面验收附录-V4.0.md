# 竞赛通-页面验收附录-V4.0

## 1. 文档定位

- 本附录用于补充 V4.0 关键页面的验收口径，覆盖业务端、H5 管理员端、PC 后台管理端。
- 本附录聚焦“开发能否交付”和“测试能否验收”，不是视觉稿替代物。

## 2. 通用验收规则

### 2.1 列表页

1. 必须支持条件筛选、分页、重置、空态、异常态。
2. 金额字段统一两位小数，积分与成长值为整数。
3. 状态字段必须使用统一枚举与字典，不允许页面自定义文案导致含义漂移。
4. 支持导出的列表必须保证“页面筛选条件”和“导出结果条件”一致。

### 2.2 详情页

1. 必须展示基础信息、状态、时间轴、关联单据、操作记录。
2. 详情页金额需能追溯到明细分摊，不允许只展示订单头汇总。
3. 涉及附件的详情页必须支持原文件名、预览、下载、失效提示。

### 2.3 配置页

1. 必须支持启停、生效时间、失效时间、复制配置。
2. 需校验重复配置冲突和生效区间重叠。
3. 所有配置变更必须写操作日志。

### 2.4 审核与财务页

1. 审核动作必须带审核意见，驳回原因必填。
2. 打款、开票、发货、核销等落账动作必须支持上传附件或填入单号。
3. 同一业务在 PC 后台和 H5 管理员端必须共用同一状态机和同一结果。

## 6. 交付验收清单

1. 页面字段、状态、按钮权限必须与主 PRD、后台 PRD、状态机附录保持一致。
2. 所有关键列表页必须完成空态、错误态、无权限态验收。
3. 所有财务页必须完成金额校验、重复点击幂等、导出一致性验收。
4. 所有上传附件页必须完成格式、大小、失效链接、预览失败验收。
5. 所有营销、权益、风控、合同、开票页面必须完成操作日志验收。
