## 2. 项目背景

竞赛通定位为面向学生、家长、老师、机构及赛事组织方的一站式竞赛服务平台，当前以**文化艺术类赛事**为主要运营方向。

**代表性赛事：**
- **安徒生(国际)艺术奖**（国际级，主打赛事，¥180/人，绘画/书法/摄影/手工）
- **"鸟巢杯"文化艺术活动**（全国级，线下，¥150/人）
- **人民艺术全国展演活动**（全国级，免费，线上+线下）

**V3.0 新增方向：**
- **OpenMAIC AI 定制课程**：老师/机构上传教学材料或输入主题，由 OpenMAIC（清华大学开源，独立部署）自动生成 AI 互动课件，学生进入 AI 课堂由多智能体（AI教师/助教/同学）实时讲解答疑。
- **H5 管理员模块**：独立登录入口（账号密码+图形验证码），支持移动端高频审核操作（身份认证/报名材料/对公转账凭证审核，公告发布，成绩发布，用户管理，AI课程审核）。

## 3. 产品目标

### 3.1 业务目标
- 建立统一的文化艺术赛事信息展示与报名入口，提升报名转化率。
- 沉淀用户、报名、订单、成绩、证书等核心数据。
- 支撑学生自主报名、老师代报名、机构代报名等多业务模式。
- **V3.0 新增**：通过 AI 定制课程差异化竞争，提供"赛事+AI课程"完整学习生态。
- **V3.0 新增**：H5 管理员模块解放 PC 依赖，提升移动端审核效率。

### 3.2 用户目标
- 学生：找赛事 → 报名 → 查进度 → 查成绩 → 下载证书 → 学习 AI 课程。
- 老师：管学生 → 代报名 → 统一支付 → 查数据 → **创建 AI 定制课程**。
- 管理员：**手机端审核** → 发布公告/成绩 → 管理用户/AI课程。

### 3.3 一期目标（V3.0 全部完成，50页原型）
- 用户端完整闭环：浏览→报名→支付→审核→成绩→证书。
- 老师端代报名+统一支付+AI课程创建与发布。
- 学生端 AI 互动课堂（OpenMAIC 独立部署接入）。
- H5 管理员端高频审核功能全覆盖。
- 双独立登录入口，安全隔离用户与管理员账号。

## 4. 用户角色

### 4.1 C 端用户
- **学生**：浏览赛事、自主报名、查看进度、查询成绩、查看下载证书、学习普通课程、学习 **AI 互动课程**（老师定制）、管理与老师绑定关系。
- **家长**：协助学生完成报名缴费、资料上传与进度跟踪。

### 4.2 业务角色
- **老师**：绑定学生（一对一）、代报名、统一支付、查看订单详情、查看报名数据统计、**创建 AI 定制课程**（上传材料/输入主题 → OpenMAIC 生成 → 预览编辑 → 发布）。
- **机构**：管理旗下老师及学生（层级管理）、批量代报名、统一支付、查看数据、**创建 AI 定制课程**。
- **统一说明**：老师与机构共用同一套 H5，通过个人中心双视角 Tab 统一管理。

### 4.3 平台角色（H5 管理员端）
- **超级管理员**：全部管理功能，含用户管理、AI课程审核。
- **审核员**：身份认证审核、报名材料审核、对公转账凭证审核。
- **运营员**：公告发布、赛事上下架、成绩发布。
- **财务员**：对公转账凭证审核、退款处理。

> 管理员账号复用 RuoYi `sys_user` 体系，`sys_role`+`sys_menu` 精确权限控制。

## 5. 使用场景

**用户端（学生/老师）：**
1. 学生或家长按艺术/音乐/舞蹈/美术等分类找近期竞赛。
2. 查看安徒生艺术奖详情，确认参赛类别和组别后完成报名（含作品电子稿上传）。
3. 老师生成绑定二维码，学生扫码绑定，由老师统一代报名支付（含对公转账）。
4. 老师上传水彩课教材 PDF，OpenMAIC 自动生成含幻灯片+测验+AI白板的互动课件，发布给绑定学生学习。
5. 学生在"我的课程→AI课堂"Tab 进入 AI 互动课堂，由 AI 教师/助教/同学协作讲解，实时答疑，完成测验后同步进度。
6. 赛后查询成绩、下载证书（JPG/PDF），或扫描证书二维码跳转公开验真页。

**管理员端：**
7. 管理员手机端查看待处理任务（认证3件/报名12件/凭证5件），逐条打开大图审核材料，一键通过或选择驳回原因。
8. 赛事成绩导入后，手机端确认发布，即时推送通知给全部参赛学生。
9. 紧急赛事变动时，手机端快速发布置顶公告。

## 7. 核心流程

### 7.1 学生自主报名流程
```
首页（Banner/推荐赛事）
 → 赛事列表（按艺术/音乐/舞蹈等分类筛选）
 → 赛事详情（安徒生艺术奖）
 → 查看参赛类别/组别/时间/费用/奖项
 → 登录账号
 → 报名流程（填信息→选类别→上传作品→微信支付）
 → 等待评审
 → 我的报名（查进度）→ 我的成绩 → 我的证书（下载/验真）
```

### 7.2 老师/机构代报名流程
```
个人中心 → 老师视角 Tab → 完成身份认证
 → 老师工作台 → 生成绑定二维码
 → 学生扫码绑定
 → 学生管理 → 选择学生 → 代报名
 → 填写学生报名信息
 → 统一支付（微信/对公转账）
 → 订单详情（查看学生信息/凭证/进度时间轴）
 → 报名数据统计（整体数据看板）
```

### 7.3 老师创建 AI 定制课程流程（V3.0新增）
```
老师工作台 → AI课程横幅/宫格「AI课程」→ maic-create.html
 → 选择输入方式（主题文字/上传PDF/Word/PPT）
 → 配置目标学段（幼/小/初/高）+ 场景类型（幻灯片/测验/AI白板/项目实践）
 → 点击「AI生成课件」
 → 生成进度动画（解析→大纲→场景→测验→优化，约30-60秒）
 → 自动跳转 maic-detail.html（预览场景列表）
 → 老师调整场景顺序/编辑测验题目
 → 设置可见范围（仅绑定学生/公开）+ 定价
 → 提交发布 → 管理员审核（admin-maic.html）
 → 审核通过 → 学生在「我的课程→AI课堂」Tab 可见
```

### 7.4 学生进入 AI 课堂流程（V3.0新增）
```
我的课程 → AI课堂 Tab → 课程卡片「进入课堂」
 → maic-classroom.html（过渡页）
 → 查看进度/AI智能体阵容
 → 点击「进入AI课堂」
 → 5步连接动画（验权→SSO Token→连接服务器→加载场景→初始化智能体）
 → 跳转 OpenMAIC 课堂（独立部署于 /maic/ 路径，SSO单点登录）
 → AI教师/助教/同学实时讲解+互动+测验
 → 课堂结束 → Webhook 回调同步进度到竞赛通
 → 返回 maic-classroom.html 查看成绩/下次学习建议
```

### 7.5 管理员审核流程（V3.0新增）
```
login.html 底部「🛡️管理员登录」→ admin-login.html
 → 账号密码+图形验证码（连续失败3次锁定30秒）
 → admin.html（工作台，待处理4格：认证/报名/凭证/成绩）
 → 点击任意模块 → 进入审核列表页
 → 点击卡片 → 进入详情页（大图查看材料）
 → 操作：通过/驳回（驳回需选原因）
 → 系统自动通知申请人（RabbitMQ消息队列）
 → 操作记录写入 sys_oper_log
```

### 7.6 证书查验流程
```
我的证书 → 查看证书 → 下载（JPG/PDF）
           → 验真 → cert-verify.html（公开，无需登录，扫码直达）
```

### 9.2 双登录入口（login.html 改造，V3.0）

### 9.6 OpenMAIC AI 定制课程模块（V3.0新增，6页）

#### 核心设计原则
- OpenMAIC **独立部署**于同一服务器（`:3100`），通过 Nginx 反向代理挂载在 `/maic/*`。
- 竞赛通后端与 OpenMAIC 通过 REST API 通信，通过 **JWT SSO Token** 实现单点登录。
- 学生学习进度、测验结果通过 **Webhook 回调**实时同步回竞赛通数据库。
- AI 大模型推荐对接**通义千问**（DASHSCOPE），数据不出境，合规。

## 11. 非功能需求

- 首页首屏加载时间 ≤ 3 秒。
- 列表分页加载响应 ≤ 2 秒。
- 高峰期支持并发报名，避免重复支付（Redis分布式锁）。
- 个人信息加密存储（身份证 AES-256）与脱敏展示。
- 优先适配微信内置浏览器（WebView），同时支持 Safari/Chrome。
- 证书下载生成 ≤ 10 秒（异步生成，轮询状态）。
- OpenMAIC 连接超时 ≤ 5 秒，超时显示友好错误提示。
- **横滑列表使用 `padding-left/right` 控制边距，禁止使用负 margin**。
- input/textarea font-size ≥ 16px，防止 iOS 自动缩放。
- 管理员 H5 会话超时 ≤ 30 分钟（前端 idle 检测）。
- 管理员图片审核：OSS 私有 Bucket + 临时签名 URL（2小时有效）。
