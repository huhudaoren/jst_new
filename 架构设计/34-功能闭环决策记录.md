## 1. 决策清单（Q-01 ~ Q-14）

| # | 问题 | 决策 | 说明/实现约束 |
|---|---|---|---|
| **Q-01** | 渠道方能否自助解绑已绑定学生？ | ✅ **可以自助解绑** | ⚠️ 必须增加 **倒计时确认弹窗**（5 秒倒计时 + 再次点击确认），防止误操作。解绑后该学生可被其他渠道方绑定。不回溯历史返点（已计提的不退）。 |
| **Q-02** | 渠道认证被驳回后重提次数限制？ | ✅ **限制 3 次** | 3 次驳回后状态置为 `locked_for_manual_review`，前端禁用"重新提交"按钮，提示"请联系客服人工审核"。管理员后台可手动解锁。 |
| **Q-03** | 临时参赛档案合并到正式账号后前端展示？ | ✅ **完全合并 + 订单标注** | 合并后订单/成绩/证书全部归到正式账号。订单详情卡顶部显示 `📎 此订单原以"张小明（临时档案）"名义报名` 标签。合并逻辑走 `participant_user_map` 表。 |
| **Q-04** | H5 管理员端与 Web 后台账号模型？ | ✅ **同账号，权限隔离，Token 层隔离** | 同一个 sys_user，通过角色区分 Web/H5 能力。Login 接口参数 `client=web\|h5` 签发不同 scope 的 Token。H5 Token 不可访问 `@PreAuthorize("hasAuthority('web:xxx')")` 的接口。 |
| **Q-05** | 渠道方认证通过时权益如何发放？ | ✅ **自动按初始等级发放基础权益** | 认证通过 → 触发器创建用户等级记录（默认最低级） → 按等级权益配置自动发放基础权益（场地减免等）到 `jst_user_rights`。后台可手动补发额外权益。 |
| **Q-06** | AI 课程创建权限范围？ | ✅ **仅渠道方 + 平台管理员** | 学生只能学习，不能创建。接口 `POST /maic/course/create` 权限点 `@PreAuthorize("hasRole('jst_channel') or hasRole('jst_platform_op')")`。 |
| **Q-07** | 团队预约未核销人数如何退款？ | ⚡ **可配置 + H5 标记 + 后台财务特批** | **重大变更**：预约退款需要在**赛事级或场次级**配置 `allow_refund` 开关。有的活动完全不支持退款（默认值），有的支持按比例退款。H5 管理员端现场标记未核销 → 后台财务按配置规则人工特批。涉及 DDL 新增字段（见 §3）。 |
| **Q-08** | 赛事方之间数据隔离？ | ✅ **完全隔离** | 赛事方 A 仅能查询 `organizer_id = A.organizer_id` 的数据。后端实现 `@PartnerDataScope` 切面，强制注入 WHERE 过滤条件。禁止跨赛事方查询。 |
| **Q-09** | 优惠券能否与积分同时使用？ | ✅ **可同时使用**（保留当前规则） | 计算顺序：标价 → 减优惠券 → 减积分抵扣 → 得净实付。返点基准 = 标价 - 平台服务费（不受优惠券/积分影响）。 |
| **Q-10** | 渠道方能否修改已绑学生档案？ | ✅ **不可自助修改** | 仅平台管理员可改。渠道方若发现错误，提交工单（jst-message 触发），平台复审后改动并留痕。 |
| **Q-11** | Web 后台权限角色 MVP 数量？ | ✅ **3 角色** | MVP 阶段先实现：① `jst_platform_op` 超级管理员（全量）② `jst_platform_auditor` 审核员（审核类菜单）③ `jst_platform_finance` 财务员（财务类菜单）。后期再扩展其他角色。 |
| **Q-12** | 成绩发布后能否修改？ | ✅ **仅平台可改，赛事方需申请** | 发布后赛事方只读。赛事方通过工单申请更正，平台审核后由管理员直接改并留痕（生成新证书，原证书标记"已作废"但保留记录）。 |
| **Q-13** | 虚拟商品能否转赠？ | ✅ **不允许转赠** | 兑换后绑定到 user_id，不可转移。禁用理由：防止权益倒卖带来的平台声誉风险。 |
| **Q-14** | 积分兑换是否支持多级？ | ✅ **仅一级**（保留当前）| 积分 → 商品一级兑换。不支持积分 → 虚拟券 → 实物的多级转换。 |

- **技术栈**：保留 RuoYi 原生 **Vue 2 + Element UI**（与 RuoYi-Vue 3.9.2 框架匹配）

## 3. 涉及 DDL 变更

### 3.1 Q-07 配置化：预约退款开关

**新增字段**：`jst_contest.allow_appointment_refund TINYINT(1) NOT NULL DEFAULT 0`

- `0` = 不允许预约退款（默认）
- `1` = 允许退款（由后台财务按比例人工处理）

**可选扩展**：若需场次级细粒度控制，另加 `jst_appointment_session.allow_refund TINYINT(1) NULLABLE`（NULL 继承赛事级）

**迁移脚本**：`架构设计/ddl/98-migration-appointment-refund-config.sql`（幂等 ADD COLUMN IF NOT EXISTS）

### 3.2 Q-02 配置化：渠道认证重提次数

**新增字段**：`jst_channel_auth_apply.reject_count INT NOT NULL DEFAULT 0`

- 每次驳回 +1
- >= 3 时不允许重提

**迁移脚本**：并入 E-0 卡

### 3.3 Q-11 新角色
- 新增 `sys_role` 行：`jst_platform_auditor` / `jst_platform_finance`
- 合并到已有 `97-migration-baseline-roles.sql` 或新开 `99-migration-phase-e-roles.sql`
