# 30. 接口测试指南（IntelliJ HTTP Client）

> 测试文件位置：`D:\coding\jst_v1\test\api-tests.http`
> **强烈推荐用 IntelliJ 自带的 HTTP Client，无需安装任何软件**

---

## 1. 为什么不用 Postman

| 维度 | IntelliJ HTTP Client | Postman |
|---|---|---|
| 安装 | ✅ IDEA 自带 | ❌ 需下载安装 |
| 文件版本管理 | ✅ .http 纯文本可入 git | ❌ 需手动导出 collection |
| 团队同步 | ✅ git pull 即更新 | ❌ 需共享 workspace |
| 学习成本 | ✅ 5 分钟 | ❌ 1 小时 |
| 跑单个请求 | ✅ 左侧绿色三角点击 | ✅ |
| 跑整个文件 | ✅ 顶部 Run All | ✅ |
| 变量提取 | ✅ JS 脚本（如 token） | ✅ |
| 断言 | ✅ client.test | ✅ |

**结论**：IntelliJ HTTP Client 完全够用。

## 2. 如何使用（5 步）

### 步骤 1：启动后端

```bash
cd D:\coding\jst_v1\RuoYi-Vue
mvn -pl ruoyi-admin -am spring-boot:run
# 或在 IDEA 中找到 RuoYiApplication 类右键 Run
```

等待控制台出现 `Started RuoYiApplication in xx seconds`，端口 8080。

### 步骤 2：在 IDEA 打开测试文件

`File → Open` 选择 `D:\coding\jst_v1\test\api-tests.http`

或在左侧 Project 面板里直接双击。

### 步骤 3：跑单个请求

每个 `###` 块的左侧栏会出现一个**绿色三角形** ▶️，点击它就执行该请求。

底部会弹出 **Services** 工具窗，显示：
- Request 详情（URL/Header/Body）
- Response 状态码 + body
- 测试结果（绿色✓ / 红色✗）

### 步骤 4：跑整个文件

点击文件**最顶部**的「Run All Requests in File」（文件第一行左上角的 ▶️）。

会按顺序执行所有请求。**这是日常回归测试方式**。

### 步骤 5：看测试断言结果

每个请求执行后，底部 Services 面板会显示：

```
✓ 登录成功 (Passed)
✓ 状态码应为 200
✓ token 不能为空
✓ userId 应为 1001
```

如有失败：

```
✗ 认领成功 (Failed)
   AssertionError: 业务码应为 200, 实际 30001
```

## 3. 测试文件结构说明

```http
@baseUrl = http://127.0.0.1:8080         # 顶部声明全局变量

### 标题: 业务名称
### 期望: 中文描述
POST {{baseUrl}}/path                     # 请求方法+URL
Content-Type: application/json
Authorization: Bearer {{auth_token}}     # 引用之前提取的变量

{
  "code": "MOCK_1001"                    # 请求体
}

> {%                                      # JS 后置脚本(响应处理 + 断言)
client.global.set("auth_token", response.body.data.token);
client.test("登录成功", function() {
    client.assert(response.status === 200);
});
%}

### --- 下一个请求 ---
```

## 4. 关键概念：变量传递

很多接口需要前一步的 token：

```http
### 1. 登录,提取 token 存入全局变量
POST {{baseUrl}}/jst/wx/auth/login
{ "code": "MOCK_1001" }

> {%
client.global.set("auth_token", response.body.data.token);
%}

### 2. 后续请求自动用 token
GET {{baseUrl}}/jst/wx/user/profile
Authorization: Bearer {{auth_token}}
```

**注意**：变量在**同一个 .http 文件内顺序执行才有效**。如果你跳着跑某个中间请求，前面的变量未填充，会失败。**建议总是从文件顶部开始 Run All**。

## 7. 常见问题排查

### Q1：响应是 401 Unauthorized
**原因**：token 失效或没传
**修复**：从顶部重新跑 A1 登录，让 `auth_token` 变量重新填充

### Q2：响应是 500 Internal Server Error
**原因**：后端异常
**修复**：看 IDEA 启动 ruoyi-admin 的控制台日志，找 stack trace

### Q3：响应是 200 但 code != 200
**原因**：业务异常（如档案不存在/状态非法跃迁/越权）
**判断**：看 `msg` 字段，常见：
- `"非法档案认领状态跃迁: ..."` → SM-14 状态机校验生效（正常拒绝）
- `"无权访问该资源"` → SecurityCheck/越权防御生效
- `"操作过于频繁,请稍后再试"` → Redisson 锁未释放，等几秒重试

### Q4：测试 B5 失败 (越权防御)
**原因**：MOCK_1002 未正确创建用户
**修复**：A1 之后跑一次 A4 用 MOCK_99999 创建一个新用户验证

### Q5：跑测试时 IDEA 报「No HTTP Client environment selected」
**修复**：文件顶部点击右侧的「Add Environment」→ 选 `dev` 即可。或忽略，全局变量也能工作。

## 8. 备选方案：Postman

如确实想用 Postman：

1. 在 Postman 中 `File → Import` → 选「Raw Text」
2. 把 .http 文件内容粘进去
3. Postman 会自动转换为 collection
4. 但不支持 IntelliJ 的 `> {% %}` JS 脚本，需手动改成 Postman 的 Tests 标签页 JS

**强烈不建议**这条路线，工作量大且文档分裂。

## 9. CI 集成（可选，未来）

