# 竞赛通 V4.1 数据模型提取报告

> 来源：《竞赛通-产品需求文档-统一版-V4.1》《竞赛通-管理后台Web端需求文档-V4.0》
> 用途：作为 25-40 张 MySQL 5.7 业务表 DDL 编写依据。
> 排除：RuoYi 内置 sys_user / sys_role / sys_menu / sys_dict / sys_log 等基础表。
> 通用约定：所有表默认含 `id BIGINT UNSIGNED PK AUTO_INCREMENT`、`create_time DATETIME`、`update_time DATETIME`、`create_by VARCHAR(64)`、`update_by VARCHAR(64)`、`deleted TINYINT DEFAULT 0`。下文不再重复列出。
> 字符集：`utf8mb4 / utf8mb4_unicode_ci`；金额字段统一 `DECIMAL(12,2)`；积分/成长值统一 `BIGINT`；JSON 字段使用 `JSON`（MySQL 5.7 原生支持）。

## 状态机汇总（State Machines & Transitions）

### SM-1 订单 order_main.order_status
- 节点：`created → pending_pay → paid → reviewing → in_service → aftersale → completed`；旁路：`cancelled / closed`
- 转换：
  - created → pending_pay：提交订单
  - pending_pay → paid：支付成功（含渠道代付）
  - pending_pay → cancelled：用户取消 / 超时
  - paid → reviewing：进入资料/凭证审核
  - reviewing → in_service：审核通过
  - reviewing → cancelled：审核驳回退款
  - in_service → aftersale：发起售后
  - aftersale → in_service：售后关闭
  - in_service → completed：售后截止且无进行中售后（系统自动）
  - completed → closed：归档

### SM-2 退款 refund_record.status
`pending → approved → refunding → completed`；旁路：`rejected / closed`
- pending → approved（审核通过）→ refunding（执行退款）→ completed
- pending → rejected（驳回）→ closed
- 资产回退顺序：现金 → 积分 → 优惠券（仅整单全退且原券有效期内回退）

### SM-3 渠道认证 channel.auth_status / channel_auth_apply.apply_status
`pending → approved`；`pending → rejected`；`approved → suspended`（启停）

### SM-4 赛事方入驻 event_partner_apply.apply_status
`draft → pending →（approved | rejected | supplement）`；supplement → pending（重提）

### SM-5 赛事 contest.audit_status / status
- 审核：`draft → pending →（approved | rejected）→ online → offline`
- 业务：`not_started → enrolling → closed → scoring → published → ended`

### SM-6 报名 enroll_record.audit_status
`pending →（approved | rejected | supplement）`；supplement → pending

### SM-7 转账凭证 payment_record.voucher_audit_status
`pending →（approved | rejected）`；rejected → pending（重传）

### SM-8 渠道返点台账 rebate_ledger.status
`pending_accrual → withdrawable → in_review → paid`；旁路：`rolled_back / negative`
- 计提条件：售后截止 + order_status=completed + list_amount>0
- 已计提未打款 → rolled_back（回退）
- 已打款发生退款 → negative（负向台账，下次提现优先抵扣）

### SM-9 渠道提现 rebate_settlement.status
`pending → reviewing →（rejected | approved）→ paid`
- approved 时系统自动扣减 negative 台账，不足部分结转

### SM-10 赛事方结算 event_settlement.status
`pending_confirm → reviewing →（rejected | pending_pay）→ paid`

### SM-11 个人预约主状态 appointment_record.main_status
`unbooked → booked →（partial_writeoff | fully_writeoff | partial_writeoff_ended | cancelled | expired | no_show）`

### SM-12 核销子项 appointment_writeoff_item.status
`unused →（used | expired | voided）`；每个子项独立流转，不联动其他子项

### SM-13 团队预约 team_appointment.status
`booked → partial_writeoff → fully_writeoff`
- 场次结束分支：0 核销 → no_show；部分 → partial_writeoff_ended；全核 → fully_writeoff
- 旁路：`cancelled / expired`

### SM-14 临时档案认领 participant.claim_status
`unclaimed →（auto_claimed | pending_manual）→ manual_claimed`
- pending_manual 仅手机号命中或多候选，进入管理员人工池
- 任何节点支持管理员撤销 → 回到 unclaimed

### SM-15 学生-渠道绑定 student_channel_binding.status
`active →（expired | replaced | manual_unbound）`
- 新绑定生效 → 旧记录置 replaced

### SM-16 优惠券 user_coupon.status
`unused → locked → used`；超时 → `expired`；退款 → `refunded`

### SM-17 用户权益 user_rights.status
`available → locked → used`；超时 → `expired`；管理员回收 → `revoked`

### SM-18 兑换订单 mall_exchange_order.status
`pending_pay → paid → pending_ship → shipped → completed`；旁路 `aftersale / closed`

### SM-19 成绩 score_record.publish_status
`unpublished → published → withdrawn`；audit_status：`pending →（approved | rejected）`

### SM-20 证书 cert_record.issue_status
`pending → issued → voided`（特批退款联动作废）

### SM-21 课程审核 course.audit_status
`draft → pending →（approved | rejected）`；status：`on / off`

### SM-22 合同 contract_record.status
`draft → pending_sign → signed →（expired | archived）`

### SM-23 发票 invoice_record.status
`pending_apply → reviewing → issuing → issued →（red_offset | voided）`

### SM-24 风险工单 risk_case.status
`open → assigned → processing → reviewing → closed`

### SM-25 表单模板 enroll_form_template.audit_status
`draft → pending →（approved | rejected）`；版本变更不影响历史 enroll_record.form_snapshot_json
