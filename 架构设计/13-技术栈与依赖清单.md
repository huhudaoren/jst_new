**SB4 兼容性风险与应对**
- Redisson 3.41.0 官方对 SB3 兼容，对 SB4 需实测；如遇 autoconfigure 失败，临时改用 `redisson` 裸 jar + 手写 `@Bean RedissonClient`，保留逻辑不变。
- 阿里云 SDK 与 Spring 解耦，无 SB 版本风险。
- wechatpay-java 0.2.x 是普通 jar，无 starter，由我们自己写 `WxPayService` 抽象，风险可控。

## 3. 模块依赖关系总览

```
ruoyi-admin (启动入口)
   ├─→ ruoyi-framework / ruoyi-system / ruoyi-quartz / ruoyi-generator (若依原生)
   └─→ jst-user / jst-event / jst-order / jst-channel / jst-points /
       jst-organizer / jst-marketing / jst-message / jst-risk / jst-finance

所有 jst-* 业务模块 ─→ jst-common ─→ ruoyi-common
                                        └─→ Redisson / aliyun-oss

jst-event ─→ aliyun-vod
jst-order ─→ wechatpay-java
```

**跨业务模块的强引用规则**：当 jst-order 的某个 Service 必须 `@Autowired` jst-user 的 `UserService` 时，**才**在 jst-order 的 `pom.xml` 增加 `<dependency>jst-user</dependency>`。**禁止预先添加未使用的依赖**。

## 4. 配置文件落点

### 4.1 application.yml 新增段（在 `ruoyi-admin/src/main/resources/application.yml`）

```yaml
# ============= 竞赛通业务配置 =============
jst:
  # 阿里云 OSS（文件上传：图片、凭证、合同）
  oss:
    endpoint: ${JST_OSS_ENDPOINT:oss-cn-hangzhou.aliyuncs.com}
    access-key-id: ${JST_OSS_AK:}            # 通过环境变量注入，禁止入库
    access-key-secret: ${JST_OSS_SK:}
    bucket: ${JST_OSS_BUCKET:jst-prod}
    base-url: ${JST_OSS_BASE_URL:https://jst-prod.oss-cn-hangzhou.aliyuncs.com}

  # 阿里云 VOD（课程视频点播）
  vod:
    region-id: ${JST_VOD_REGION:cn-shanghai}
    access-key-id: ${JST_VOD_AK:}
    access-key-secret: ${JST_VOD_SK:}

  # 微信支付（决策 Q4=B：本期 mock，profile=prod 切真实）
  wxpay:
    enabled: ${JST_WXPAY_ENABLED:false}      # 默认 mock；prod 环境置 true
    mch-id: ${JST_WXPAY_MCHID:}
    app-id: ${JST_WXPAY_APPID:}
    api-v3-key: ${JST_WXPAY_APIV3:}
    private-key-path: ${JST_WXPAY_PRIVKEY:}
    notify-url: ${JST_WXPAY_NOTIFY:https://api.jst.example.com/jst/pay/notify/wechat}

  # 业务参数
  biz:
    aftersale-default-days: 7         # 售后宽限默认天数
    snowflake-worker-id: 1            # 雪花 ID worker
    snowflake-datacenter-id: 1
    points-cash-rate: 100             # 100 积分=1 元（PRD 定义）

# Redisson（决策 Q1=A）
spring:
  data:
    redis:
      redisson:
        config: |
          singleServerConfig:
            address: "redis://${spring.data.redis.host}:${spring.data.redis.port}"
            password: ${spring.data.redis.password:}
            database: ${spring.data.redis.database:0}
```

## 5. 微信支付占位实现规范（决策 Q4=B）

`jst-order` 内必须存在如下三件套：

```java
// 1. 抽象接口
package com.ruoyi.jst.order.pay;
public interface WxPayService {
    PrepayResVO unifiedOrder(PrepayReqDTO req);
    void handleNotify(String body, Map<String,String> headers);
    RefundResVO refund(RefundReqDTO req);
}

// 2. Mock 实现（默认启用，开发期可跑通完整下单流程）
@Service
@ConditionalOnProperty(prefix="jst.wxpay", name="enabled", havingValue="false", matchIfMissing=true)
public class MockWxPayServiceImpl implements WxPayService { ... }

// 3. 真实实现（生产环境启用）
@Service
@ConditionalOnProperty(prefix="jst.wxpay", name="enabled", havingValue="true")
public class RealWxPayServiceImpl implements WxPayService { ... }
```

## 6. 禁止引入的依赖（避免技术栈膨胀）

| 禁止 | 原因 | 替代方案 |
|---|---|---|
| Lombok | 若依 4.x 默认未引入，保持团队习惯一致 | 手写 getter/setter |
| MyBatis-Plus | 与若依原生 MyBatis 冲突 | 用若依封装的 BaseEntity + 通用 Mapper |
| XXL-JOB | 决策 Q2=Quartz | 使用 ruoyi-quartz |
| Hutool | 若依内置 utils 已足够 | StringUtils/DateUtils（若依已封装） |
| Spring Cloud / Nacos | 单体应用 | 若有需要单独立项 |
| Knife4j | 已使用 springdoc | 沿用 springdoc |
